# v4.6.15 — Checklist list service boundary

## Summary

**v4.6.15** introduces a minimal **service layer** for the read-only `checklist list` command, keeping existing CLI human output, JSON shape (`Vec<ChecklistItem>`), ordering, checked/unchecked state, and Trip not-found behavior unchanged while making the use case reusable for future Tauri GUI screens.

Specification: [v4.6.15-checklist-list-service-boundary.md](../specifications/v4.6.15-checklist-list-service-boundary.md)

## What changed

- New `src/services/checklist_list.rs` — `list_checklist()` returns `ChecklistListServiceResult` (no terminal I/O)
- `checklist list` CLI handler calls service, then routes to existing human/JSON output

## Compatibility

- `checklist list` human output unchanged (header + `[x]` / `[ ]` marks)
- `checklist list --json` unchanged (`Vec<ChecklistItem>` serialization)
- Ordering unchanged (`ORDER BY is_done ASC, sort_order ASC, id ASC`)
- checklist auto-generate / write commands unchanged
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No `checklist show` / write command changes
- No auto-generate changes
- No Tauri / GUI implementation
