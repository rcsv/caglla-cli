# v3.7.1 — Okinawa Receipt Inbox sample and export fix

## Summary

**v3.7.1** is a **patch release** after v3.7.0. It adds the Okinawa Receipt Inbox sample story and fixes export formatting for trashed Receipt timestamps so **`validate-export` accepts exported trashed Receipts**.

Extends the Okinawa / Sesoko canonical sample with a **Receipt Inbox workflow story** (assign / trash / pending sum). **Pending Receipt sum is not Actual** — only Expenses created by **`receipt assign`** enter Actual spending.

Requires **v3.7.0** baseline. Current trip export still writes **`schema_version: 8`**.

Design series:

```text
v3.7.0 Receipt assignment and trash workflow  → v3.7.0-notes.md
Canonical sample Receipt Inbox extension      → feb4043
Release v3.7.1                                → this release
```

### Terminology

| Term | Meaning |
|---|---|
| **Receipt** | Unassigned payment candidate in the Inbox (not Actual) |
| **Pending Receipt sum** | Total of active Receipts by currency — **not** Actual spending |
| **Trash** | Receipt with `trashed_at` set — excluded from active Inbox |
| **Expense** | Confirmed actual spending — only created by `receipt assign` |

## Fixed

- **Trashed Receipt export timestamp** — `trip export` now emits **RFC3339** for `receipts[].trashed_at` when the DB stores SQLite naive datetime (`YYYY-MM-DD HH:MM:SS`). Previously exported the raw DB string, which could fail `validate-export` RFC3339 checks on exports that include trashed Receipts.
- Implementation: `format_db_timestamp_for_export` in `src/receipt.rs` (pass-through if already RFC3339; convert naive local datetime otherwise).

## Notes

The Okinawa sample illustrates the intended **post-trip Receipt cleanup** flow: register loose receipts during travel, review at home, **assign** to Itineraries as Expenses or **trash** personal items.

`receipt assign` is **not** a link — it atomically creates an Expense and removes the Receipt from the Inbox.

## Upgrade

1. Replace binary with v3.7.1 build (or `cargo build --release` from tag `v3.7.1`).
