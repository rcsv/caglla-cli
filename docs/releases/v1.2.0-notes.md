# v1.2.0 Day commands

## Summary

- Day is now a first-class CLI entity: `day list`, `day show`, and `day swap`.
- Day dates are derived from `trip.start_date + (day_number - 1)`; no `date` column on `days`.
- `day list` and `day show` support `--json` for tool integration.
- Day creation/deletion remain automatic via Trip create/update (`day add` / `day delete` are not implemented).

## Highlights

- **`day list <trip_id>`** — Trip name plus each Day with derived calendar date
- **`day show <trip_id> <day_number>`** — Day date and Itineraries sorted like timeline (`day` → `start_time` → `sort_order`)
- **`day swap <trip_id> <day_a> <day_b>`** — Swap Itinerary items between two Days inside a DB transaction
- **Fixed day numbers** — Swap exchanges content only; `day_number` and Trip dates do not change
- **Export/import unchanged** — schema v1; swap updates `day_id` and denormalized `day` column in DB only

## Commands

```bash
caglla day list 1
caglla day show 1 2
caglla day swap 1 2 3
caglla day list 1 --json
caglla day show 1 2 --json
```

## Not in this release

- `day add` / `day delete` (Day lifecycle stays on Trip create/update)
- `day update` (Day title/description editing)
- export schema v2 (`days[]`)
- Markdown Day title/description sections

## Breaking Changes

None. Existing commands and export/import behavior are unchanged.
