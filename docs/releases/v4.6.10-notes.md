# v4.6.10 — Itinerary show service boundary

## Summary

**v4.6.10** introduces a minimal **service layer** for the read-only `itinerary show` command, keeping existing CLI human output, JSON shape, and Itinerary not-found behavior unchanged while making the use case reusable for future Tauri GUI Itinerary detail screens.

Specification: [v4.6.10-itinerary-show-service-boundary.md](../specifications/v4.6.10-itinerary-show-service-boundary.md)

## What changed

- New `src/services/itinerary_show.rs` — `show_itinerary()` returns `ItineraryShowServiceResult` (no terminal I/O)
- `itinerary show` CLI handler calls service, then routes to existing human/JSON output

## Compatibility

- `itinerary show` human output unchanged (detail + optional Reservations section)
- `itinerary show --json` unchanged (`ItineraryItem` serialization)
- Itinerary not found unchanged (`Itinerary not found: <id>`)
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No new detail fields or related-data expansion in service result
- No `itinerary list` / `itinerary timeline` changes
- No Tauri / GUI implementation
