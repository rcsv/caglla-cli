# v4.6.8 — Itinerary list service boundary

## Summary

**v4.6.8** introduces a minimal **service layer** for the read-only `itinerary list` command, keeping existing CLI human/JSON output, ordering (Sequence-first), and not-found behavior unchanged while making the use case reusable for future Tauri GUI itinerary list screens.

Specification: [v4.6.8-itinerary-list-service-boundary.md](../specifications/v4.6.8-itinerary-list-service-boundary.md)

## What changed

- New `src/services/itinerary_list.rs` — `list_itineraries()` returns `ItineraryListServiceResult` (no terminal I/O)
- `itinerary list` CLI handler calls service, then routes to existing human/JSON output

## Compatibility

- `itinerary list` human output unchanged
- `itinerary list --json` output unchanged (serializes `Vec<ItineraryItem>`)
- Ordering unchanged (`ORDER BY day_number, sort_order, id`)
- Trip not found unchanged (`Trip not found: <id>`)
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No `itinerary show` / `itinerary timeline`
- No itinerary write commands
- No Tauri / GUI implementation
