# v1.10.0 — Travel Ledger responsibilities

## Summary

Documentation-only release. Clarifies Travel Ledger responsibilities for **Summary**, **Remark**, **Note**, and **Reservation**.

Core message:

```text
Summary     = 見せる要約
Remark      = 備考
Note entity = メモ / 詳細メモ
Reservation = 予約情報
```

## Added

- Added `docs/specifications/travel-ledger-responsibilities.md` — responsibility boundaries for auxiliary travel-ledger information.
- Documented **Summary** as shared / printable overview (Trip Summary, Day Summary — future implementation).
- Documented `itinerary_items.note` as **Remark / 備考** — short inline supplement on one itinerary row.
- Documented **Note entity** as long-form free text (planning, context, reflection — CRUD already exists).
- Documented **Reservation** as a **future structured entity**, not a Note.

## Clarified

- Trip / Day Summary are **future concepts** for guidebook and sharing views (before-trip planning), not long-form Notes.
- Reservation **belongs to Itinerary** but can be **aggregated at Trip level** for guidebook / itinerary booklet export.
- [Okinawa Sesoko canonical sample](../../samples/okinawa_sesoko_2026/) remains focused on **post-trip ledger and settlement validation**; Summary and Reservation entities are intentionally omitted from the seed data.
