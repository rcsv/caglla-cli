# v4.6.0 — TripStats.days semantics fix

## Summary

Core architecture hardening の最初の具体修正。`trip stats` の `days` が **Trip day count**（旅行日数）を表すように修正しました。

従来は itinerary の最大 `day` から算出されていたため、Itinerary が一部の日だけ登録されている Trip で日数が過小表示になることがありました。

Specification: [v4.6.0-trip-stats-days-semantics-fix.md](../specifications/v4.6.0-trip-stats-days-semantics-fix.md)

## Changed

- `TripStats.days` resolution order:
  1. `days` テーブル件数（正本）
  2. `days` が空のとき Trip `start_date` / `end_date` から算出
  3. それも不可のとき legacy fallback（itinerary max day）
- Travel Book `Trip overview` の `- Days:` も同じ意味（旅行日数）になる

**Breaking semantics note (intentional):**

```text
trip stats `days` now represents Trip day count, not max itinerary day.
```

## Unchanged

- DB schema
- JSON export/import schema
- `trip stats --json` field 名 `days`（意味のみ修正）
- Expense / Estimate / Difference / Receipt 集計
- Travel Book 章立て・本文仕様（Okinawa golden は Colophon `Version:` のみ更新）

## Non-goals

- `itinerary_day_span` 等の別指標追加
- `accommodation_nights` / Hotel 宿泊数の自動算出 / overnight flight 判定
- locale-aware duration label（「日帰り」「何泊何日」）— 仕様メモのみ、実装は defer
- SQLite FK / migration strategy
- command handler split
