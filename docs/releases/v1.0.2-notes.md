# v1.0.2 Markdown export and documentation polish

## Summary

This patch release improves Markdown file export usability and clarifies README guidance for resource IDs and JSON import/export behavior.

## Changes

- Polish `trip export-md --output` handling:
  - Separate Markdown generation from stdout/file I/O
  - Success message: `Markdown exported: <path>`
  - Add CLI integration tests for file export and stdout mode
- Clarify README guidance for Trip / Itinerary / Checklist ID usage
- Document JSON import/export limitations in README:
  - Export/import covers Trip and Itinerary only (Checklist is not included)
  - Imported Trip IDs are newly assigned; use the ID shown after import
