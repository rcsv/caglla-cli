# v1.6.0 Export Schema v3 with Expense backup

## Summary

v1.6.0 adds **Export Schema v3** support with nested **Expense export/import**.

Expense records are now included under each itinerary in the trip export JSON. This avoids relying on database IDs during export/import and preserves itinerary–expense relationships through the JSON parent-child structure (`days[]` → `itineraries[]` → `expenses[]`).

The import command supports both **schema v2** and **schema v3**. Schema v2 exports remain valid and importable. `validate-export` now checks nested Expense records.

## Added

- **Export Schema v3** — `schema_version: 3` on new exports
- **Nested Expense export** — `days[].itineraries[].expenses[]` (no top-level `expenses[]` array)
- **Expense import** — restores expenses from nested structure after Trip / Day / Itinerary creation
- **`validate-export` Expense checks** — `amount` / `currency` required; `validate_currency_code()` for currency format
- **v3 types** — `TripExportV3`, `ExportDayV3`, `ExportItineraryV3`, `ExportExpenseV3`
- **Tests** — v3 export/import/validate and roundtrip coverage

## Improved

- **Import routing** — `schema_version` に応じて v2 / v3 を自動判別
- **`load_trip_export_from_file`** — v3 を読み込み、既存処理向けに flatten 可能（`trip diff` 等の互換）
- **Import summary** — v3 import でも従来どおり表示

## Compatibility

| Topic | v1.6.0 behavior |
|---|---|
| **Current export** | `schema_version: 3`, nested `days[]` with expenses |
| **v2 import** | 引き続きサポート（Expense なし） |
| **v1 import** | 引き続きサポート（`schema_version` 未指定 / `1`） |
| **Expense fields** | `title`, `amount`, `currency`, `paid_by_name`, `expense_date`, `note`, `sort_order`（DB id は export しない） |
| **Breaking** | 新規 export の JSON 形状が v2 から変更（`itinerary_items` トップレベル → `days` ネスト） |

## Not included

- `trip diff` Expense comparison
- Participant / Settlement / Shared Expense
- Currency conversion / exchange rates
- Photo export
- XML / XSD export

## Breaking Changes

New exports use schema v3 with a different JSON shape. **Import** remains backward compatible with v1/v2 exports.
