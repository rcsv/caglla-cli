# v3.10.0 — DB Use concept design

## Summary

**v3.10.0** is a **documentation-only release**. It defines **`db use`** as the Phase 2 companion to v3.9.0 DB path resolution: a command that **writes** `[database].path` in CWD `./caglla.toml` for persistent default database selection. No CLI behavior, SQLite schema, or trip export schema changes.

Requires **v3.9.2** baseline. Trip export still writes **`schema_version: 8`**.

## Added

### DB Use concept design document

[v3.10.0-db-use-concept-design.md](../specifications/v3.10.0-db-use-concept-design.md) — canonical Phase 2 design for `db use`:

- **`db use <path>`** — update CWD `./caglla.toml` `[database].path`; does **not** open or create the DB file
- **`db use --clear`** — remove configured path and revert to default `./caglla.db` when no overrides
- **Not a runtime override** — sets the config layer for **future** invocations; `--db` and `CAGLLA_DB` still win on each run
- **Path storage** — prefer relative paths from the config directory; absolute when needed
- **Existing TOML** — merge `database.path` only; preserve unknown keys; comments not guaranteed on rewrite
- **Missing DB file** — allowed at `db use` time (same spirit as `--db` on a new file)
- **Parent-directory config search** — explicitly **deferred** beyond Phase 2 (separate milestone)

## Clarified

| Topic | Clarification |
|---|---|
| **Phase 1 vs Phase 2** | v3.9.0 = read resolution; v3.10.0 design = write via `db use` (implementation TBD) |
| **`db use` vs `--db`** | `db use` persists config; `--db` is per-invocation override |
| **Write target** | CWD `./caglla.toml` only in Phase 2 — no parent walk, no global config |
| **Comments in TOML** | Phase 2 merge-write may drop comments — documented limitation |
