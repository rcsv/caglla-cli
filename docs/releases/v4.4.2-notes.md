# v4.4.2 — Travel Book presentation helper review

## Summary

**v4.4.2** is a **documentation-only review**. It inventories `markdown.rs` Travel Book logic and classifies each concern as Domain / Presentation helper / Markdown renderer / Future GUI / Defer.

## Headline findings

- **~60% of Travel Book decisions are presentation helper candidates** — reusable beyond Markdown
- **v4.4.1 `format_travel_book_category_detail_line` is the correct extraction pattern**
- **Reservation helpers** (`reservation_provider_line_redundant`, `format_travel_book_reservation_period`, `format_travel_book_reservation_heading`) are high-priority move candidates
- **Markdown syntax** (headings, tables, checkboxes, blank lines) stays in renderer

## Recommended next step

Defer: `TravelBookDocument` full abstraction, GUI, Venue / Maps.
