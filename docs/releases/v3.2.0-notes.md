## Summary

**v3.2.0** adds **Database status foundation** вЂ” read-only visibility into which SQLite file Caglla CLI uses, without DB path switching.

Requires **v3.1.0** baseline. Current `trip export` still writes **`schema_version: 6`**.

| Term | Meaning |
|---|---|

## Added

### `caglla db path`

Prints the resolved **absolute** path to `caglla.db` (one line).

- Does **not** open SQLite
- Does **not** create the file when missing
- No `--json` in v3.2.0

```bash
caglla db path
# /absolute/path/to/caglla.db
```

### `caglla db status`

Human-readable summary:

| Field | When |
|---|---|
| Path | always (absolute) |
| Exists | always |
| File size (bytes) | file exists only |
| Table counts | file exists only (post-migration) |
| Trip export schema version | always вЂ” labeled as trip export JSON, not SQLite migration |

When the DB file **does not exist**:

- No open, no file creation
- Size and table counts omitted

When the file **exists**:

- Opens via normal `open_db_at()` flow (migrations applied)
- Reports row counts for: `trips`, `days`, `itinerary_items`, `notes`, `expenses`, `expense_beneficiaries`, `estimates`, `participants`, `reservations`, `checklist_items`

### `caglla db status --json`

Pretty JSON only on stdout (no human headings).

```json
{
  "schema_version": 1,
  "path": "/abs/path/caglla.db",
  "exists": true,
  "file_size_bytes": 204800,
  "trip_export_schema_version": 6,
  "table_counts": {
    "trips": 1,
    "days": 4,
    "itinerary_items": 12,
    "notes": 0,
    "expenses": 3,
    "expense_beneficiaries": 2,
    "estimates": 5,
    "participants": 2,
    "reservations": 0,
    "checklist_items": 8
  }
}
```

When `exists: false`, `file_size_bytes` and `table_counts` are **omitted** (not `null`).

### Dispatch change

`db path` and `db status` are handled **before** the shared `open_db()` in `main.rs`, so missing DB files are not accidentally created (unlike other commands).

### Unchanged

- **`db reset`** вЂ” same behavior (`open_db()` then wipe data)
- **DB path rule** вЂ” still CWD-relative `caglla.db`
- **Other commands** вЂ” unchanged `open_db()` flow

## Upgrade

1. Replace binary with v3.2.0 build (or `cargo build --release` from tag `v3.2.0`).
2. Run `caglla db path` from your working directory to confirm which `caglla.db` is used.
3. Run `caglla db status` before operations in unfamiliar directories.

No DB migration changes in this release.
