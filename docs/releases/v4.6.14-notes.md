# v4.6.14 — Expense list service boundary

## Summary

**v4.6.14** introduces a minimal **service layer** for the read-only `expense list` command, keeping existing CLI human output, JSON shape (`ExpenseListJson`), ordering, trip stats semantics, and not-found behavior unchanged while making the use case reusable for future Tauri GUI screens.

Specification: [v4.6.14-expense-list-service-boundary.md](../specifications/v4.6.14-expense-list-service-boundary.md)

## What changed

- New `src/services/expense_list.rs` — `list_expenses()` returns `ExpenseListServiceResult` (no terminal I/O)
- `expense list` CLI handler calls service, then routes to existing human/JSON output

## Compatibility

- `expense list` human output unchanged (table format)
- `expense list --json` unchanged (`ExpenseListJson` with `ExpenseJson` items)
- trip stats / Receipt / Estimate / Actual boundaries unchanged
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No `expense show` / write command changes
- No new totals or aggregation in service result
- No Tauri / GUI implementation
