# v1.3.0 Note model and CLI commands

## Summary

- Introduced the **Note** entity for free-form records at Trip, Day, and Itinerary levels.
- Added `note add`, `note list`, `note show`, `note update`, and `note delete` with `--json` on list/show.
- Application-level cascade deletes Notes when Trip, Day (on trip shrink), or Itinerary is removed.
- Export/import remain on **schema v1** without `notes[]`; existing `itinerary_items.note` is unchanged.

## Added

- **`notes` table** — polymorphic `owner_type` + `owner_id` (`trip` / `day` / `itinerary`)
- **`Note` / `NoteOwnerType` models** — validation and JSON serialization
- **`note` CLI** — Trip / Day / Itinerary Note CRUD
- **Day Note resolution** — CLI `--trip` + `--day` → stored as `days.id`
- **Cascade helpers** — `delete_notes_for_trip`, `delete_notes_for_day`, `delete_notes_for_itinerary`
- **Tests** — CRUD, cascade, `day swap` Note behavior, CLI integration
- **Docs** — `docs/specifications/note-model.md`, README Note section

## Improved

- **`trip delete`** — removes Trip / Day / Itinerary Notes before deleting the Trip (subquery over `days` and `itinerary_items`)
- **`trip update` (shrink)** — deletes Day Notes on removed empty Days; blocks shrink when target Day has Itinerary
- **`itinerary delete`** — removes Itinerary Notes for the deleted item only

## Notes / Compatibility

| Topic | v1.3.0 behavior |
|---|---|
| **Export schema v2** | **Not supported** — `notes[]` deferred to a future release |
| **schema v1 export/import** | Unchanged — Notes are **not** included in backup JSON |
| **`itinerary_items.note`** | **Retained** — single inline field per itinerary; coexists with Note entity |
| **Cascade** | **Application-side** — no FK on `notes`; delete paths call cascade helpers |
| **`day swap`** | Notes unchanged — Day Notes stay on `days.id`; Itinerary Notes stay on `itinerary_items.id` |
| **Transactions on delete** | Not added in this release (known follow-up for v1.3.1 / v1.4.0) |

### Not in this release

- Export schema v2 (`notes[]`)
- `trip export-md` Note sections
- `trip diff` / `validate-export` Note checks
- `trip doctor` / `trip stats` Note integration
- `note list --all`
- Editor-based body input
- Photo / Expense / Itinerary Checklist

## Breaking Changes

None. Existing commands and schema v1 export/import behave as before.
