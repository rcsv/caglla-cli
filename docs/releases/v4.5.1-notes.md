# v4.5.1 — doctor / advisor Receipt utilization

## Summary

Ledger hygiene release: `trip doctor` / `trip advisor` now detects **pending receipts** in Receipt Inbox and flags **inconsistent receipt state**.

Receipt Inbox is still **not Actual** — this change does **not** affect:

- `trip stats` totals (Expense-only Actual)
- Travel Book export (`trip export-md`)
- trip export/import schema
- SQLite schema

Implementation plan: [v4.5.1-doctor-advisor-receipt-utilization-implementation-plan.md](../specifications/v4.5.1-doctor-advisor-receipt-utilization-implementation-plan.md)

## Added

- Doctor issues:
  - `pending_receipts` (info): active receipts remain in Receipt Inbox
  - `inconsistent_receipt_state` (warning): `status` and `trashed_at` inconsistency
- Advisor advice and `--with-commands` command hints for receipt review / assign / trash / restore
- Doctor/Advisor JSON `schema_version: 2` (additive issue codes + receipt target/details)

## Non-goals

- Treat receipts as Potential Actual
- Change stats / export-md / export schema
- Receipt workflow changes (assign consumption model unchanged)
