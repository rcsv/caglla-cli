# v1.0.5 Data completeness and backup reliability

## Summary

This release improves day-to-day CLI usability for copying trips, previewing checklist generation, and backing up travel data with versioned export metadata.

## Added

### Trip Duplicate

```bash
trip duplicate <trip_id>
trip duplicate <trip_id> --name "Copy"
```

- Duplicate Trip, Itinerary, and Checklist together with newly assigned IDs
- Default name: `{original name} (Copy)`

### Checklist Generate Dry Run

```bash
trip checklist-generate <trip_id> --dry-run
```

- Preview items that would be added or skipped without modifying the database

## Improved

### Export Metadata

Export JSON now includes:

```json
{
  "schema_version": 1,
  "exported_at": "..."
}
```

- `schema_version` is `1`
- `exported_at` is the export timestamp in RFC3339

### Import Compatibility

Import remains backward compatible with:

- New format (with metadata)
- Legacy format with `checklist_items`
- Legacy format without `checklist_items`

Metadata fields are ignored during import.

### Roundtrip Validation

Added structural comparison tests for:

```text
Export → Import → Re-Export
```

Including Checklist data, to improve backup/restore reliability.
