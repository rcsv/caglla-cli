# v3.7.0 — Receipt assignment and trash workflow

## Summary

**v3.7.0** implements the **Receipt assignment and trash workflow** — assign Receipts to Itineraries as Expenses, move unwanted Receipts to Trash, restore from Trash, and show a pending Receipt summary. This is an **implementation release** (not documentation-only).

Requires **v3.6.0** baseline. Current trip export now writes **`schema_version: 8`**.

### Terminology

| Term | Meaning |
|---|---|
| **Estimate** | Planned amount before the trip |
| **Receipt** | Unassigned payment candidate in the Inbox (not Actual) |
| **Pending Receipt sum** | Total of active Receipts by currency — **not** Actual spending |
| **Expense** | Confirmed actual spending |
| **Trash** | Receipt with `trashed_at` set — excluded from active Inbox |
| **Difference** | Planned-vs-actual delta based on **confirmed** Expenses |

---

## Added

- Added `receipt assign` — transactional Receipt → Expense promotion (Receipt is **deleted** after assign).
- Added `receipt trash` — move Receipt to Trash (`trashed_at` set; not physical delete).
- Added `receipt restore` — restore trashed Receipt to active Inbox.
- Added **Pending Receipt summary** on `receipt list` / `receipt list --json` (active Receipts only).
- Added `receipts.trashed_at` column and index (`idx_receipts_trashed`).
- Added `receipt list --trashed` and `receipt list --all` filters.
- Added export schema **v8** with optional `receipts[].trashed_at`.
- Added import support for schema v8 receipts; v6 / v7 import remains supported.
- Added `validate-export` checks for `trashed_at` (RFC3339) on schema v8.
- Added `trip diff` support for `trashed_at` changes on receipts.

---

## Changed

- Export schema version is now **v8**.
- Receipt lifecycle uses **`trashed_at`** for active vs trashed:
  - `trashed_at IS NULL` — active Inbox Receipt
  - `trashed_at IS NOT NULL` — Trash
- `receipt ignore` is a **deprecated alias** for `receipt trash` (same behavior; message notes deprecation).
- Existing `status = ignored` Receipts are migrated to Trash on first run (`trashed_at` set).
- `receipt list` defaults to **active** Receipts only (Trash excluded unless `--trashed` or `--all`).
- `receipts.status` is **retained** in v3.7.0 (not dropped); `ignored` remains in export for compatibility.

---

## Compatibility

- **v6 / v7 imports** remain supported.
- v7 exports with `status: ignored` import as trashed (`trashed_at` set).
- v7 exports without `trashed_at` import as active.
- Existing **Planned vs Actual** behavior remains unchanged.
- Existing DBs receive `trashed_at` column and `ignored → trashed` normalization on first run after upgrade.
- `linked` / `converted` / `receipt link` / `linked_expense_id` are **not** reintroduced.

## Notes

`receipt assign` is **not** a link operation. It atomically creates an Expense from Receipt metadata and removes the Receipt from the Inbox.

Receipt and Pending Receipt sum remain **outside** Planned vs Actual. Only the Expense created by assign enters Actual spending.

Image evidence should remain a shared **Evidence / Attachment** layer for both Receipt and Expense in future work — not as Receipt-only `image_path`.

---

## Upgrade

1. Replace binary with v3.7.0 build (or `cargo build --release` from tag `v3.7.0`).
2. On first run, `receipts.trashed_at` is added automatically and existing `ignored` Receipts are migrated to Trash. No manual migration step.
3. `trip export` now writes `schema_version: 8` with optional `receipts[].trashed_at`.
4. v6 / v7 export files import without change.
