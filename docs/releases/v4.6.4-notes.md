# v4.6.4 — Read-only service boundary pilot

## Summary

**v4.6.4** introduces a minimal **service layer** for the read-only `trip stats` command, separating structured results from CLI terminal output in preparation for future Tauri GUI reuse.

Specification: [v4.6.4-read-only-service-boundary-pilot.md](../specifications/v4.6.4-read-only-service-boundary-pilot.md)

## What changed

- New `src/services/trip_stats.rs` — `get_trip_stats()` returns `TripStatsServiceResult` (no `println!`)
- `trip stats` CLI handler calls service, then formats human or JSON output
- `print_trip_stats_display(&TripStats)` — CLI-only formatting extracted from DB-coupled `print_trip_stats`

## Compatibility

- CLI human output unchanged
- `trip stats --json` schema unchanged (`TripStats`)
- No DB schema or migration changes
- Okinawa golden: Colophon `Version:` only

## Non-goals

- Tauri / GUI implementation
- Other read-only commands (deferred)
- Write command service split

## Next candidates

- **trip list** or **trip show** — next read-only service pilots
- migration runner / orphan detection (parallel track)
