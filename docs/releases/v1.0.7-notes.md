# v1.0.7 Backup validation and import summary

## Summary

- Added `validate-export` to check whether an exported trip file can be imported.
- Improved `trip import` output with a clearer summary.
- Kept legacy export files importable while surfacing warnings.

## Added

### trip validate-export

```bash
trip validate-export <file>
trip validate-export <file> --json
```

- Validates export JSON without using the database.
- Text output shows Checks, Summary, Warnings, and Result.
- `--json` returns structured `ExportValidationReport` with `checks[]`, `warnings`, and `errors` (always an array, including `[]`).

Key semantics:

| Field | Meaning |
|---|---|
| `valid` | File can be imported (`errors` is empty) |
| `warnings` | Differences from the recommended export format |
| `checks` | Structural checks (may be false while still `valid` for legacy files) |

Report JSON uses:

- `schema_version` — validation report format version
- `export_schema_version` — `schema_version` value inside the export file (if present)

## Improved

### trip import summary

Import success output now includes:

```text
Trip:
  <name> (ID: <new_id>)

Created:
  Itineraries / Checklist counts

Schema:
  version 1
```

Legacy imports show:

```text
Schema:
  未指定（旧形式）
```

Import logic (`import_trip_from_export`) is unchanged; only the summary display was improved.

## Compatibility

- Legacy export files remain importable.
- `validate-export` treats importable legacy files as `valid: true` with warnings (e.g. missing `schema_version` or `checklist_items` key).
- No database schema changes.
- No export format changes.
