# v1.5.0 Expense CRUD support

## Summary

v1.5.0 introduces **Expense CRUD support**.

Expenses are **itinerary-scoped** spending records with **`amount` and `currency` as required fields**.  
`title`, `note`, `paid_by_name`, and `expense_date` are optional.

Export schema v3, import, settlement, and multi-currency conversion are **not** in this release.

## Added

- **`expenses` table** — itinerary-scoped spending records
- **`Expense` model** — `src/models.rs`
- **CLI** — `expense add`, `expense list`, `expense show`, `expense update`, `expense delete`
- **`expense list --itinerary`** — expenses for one itinerary
- **`expense list --trip`** — aggregated expenses for a trip
- **`expense list --json`** / **`expense show --json`** — pretty JSON output
- **`validate_currency_code()`** — uppercase normalization + 3-letter alphabetic check (unknown codes allowed)
- **`parse_amount_for_currency()`** — string-based parsing to minor-unit integers (e.g. JPY `1500`, USD `12.50` → `1250`)
- **Cascade delete** — expenses removed on `itinerary delete` and `trip delete` (transactional)
- **Tests** — unit tests in `expense.rs`, CLI integration tests in `tests/expense_cli.rs`
- **Docs** — README Expense Management section; [expense-model.md](../specifications/expense-model.md) status update

## Changed

- **`db reset`** — clears `expenses` and resets `sqlite_sequence`
- **`itinerary delete` / `trip delete`** — also delete related expenses (alongside existing Note cascade)

## Validation

| Topic | v1.5.0 behavior |
|---|---|
| **Required** | `amount`, `currency` on add |
| **Optional** | `title`, `note`, `paid_by_name`, `expense_date` |
| **Currency** | Normalized to uppercase; 3-letter alphabetic form; unknown codes permitted |
| **Amount** | Stored as INTEGER minor units; CLI accepts decimals for currencies like USD |
| **Parent** | `add` requires `--itinerary`; `list` requires `--itinerary` or `--trip` |

## Not included

The following are **explicitly out of scope** for v1.5.0:

- Export Schema v3
- Import
- `validate-export`
- `trip diff` Expense comparison
- `trip stats` spending aggregation
- Participant
- Settlement
- Exchange rate / multi-currency conversion
- XML / XSD export

These are planned for later phases (see [expense-model.md](../specifications/expense-model.md)).
