# v1.7.0 Canonical sample dataset and expense verification

## Summary

v1.7.0 strengthens **Expense backup reliability** and adds a **real-world canonical sample dataset** (`samples/okinawa_sesoko_2026/`). The theme is verifying that the current domain model can represent actual travel events — highways, parking, rental cars, fuel, groceries, hotels, sightseeing, personal expenses, and receipts — without adding new Expense categories or Participant/Settlement models.

## Canonical Sample Dataset

**`samples/okinawa_sesoko_2026/`** — Okinawa Sesoko trip (2026-04-26〜04-29), derived from real travel data (`EstimateTrip_20260426.pdf`).

| Item | Count |
|---|---|
| Itineraries | 58 |
| Expenses | 49 |
| Total (JPY) | ¥561,780 |

Includes: highways, parking, airport, rental car, fuel, grocery shopping, hotel check-in/out, sightseeing, personal expenses (`paid_by_name`), receipt numbers in notes.

Files:

- `seed.sh` — CLI seed script (`CAGLLA_SAMPLE_WORKDIR` supported for isolated DB)
- `expected-export-v3.json` — normalized golden export
- `README.md` — seed rules and verification commands

Integration tests: `tests/okinawa_sesoko_seed_cli.rs`

## Expense Verification

### P0 — Backup stability

- **`trip duplicate`** — copies Expenses via schema v3 export/import path
- **Expense roundtrip** — export → import → re-export integration test with nested v3 expenses
- **`comparable_export_json`** — v3 `days[].itineraries[].expenses[]` comparison
- **`validate-export`** — Expense error-case tests (invalid currency, empty currency, invalid date)
- **Documentation** — `export-schema.md` and `expense-model.md` aligned with v3

### P1 — Data visibility

- **`trip export-md`** — simple `Expenses:` list under each Itinerary (data verification, not layout polish)
- **`trip stats`** — Expense count and per-currency totals (no conversion, no Settlement)
