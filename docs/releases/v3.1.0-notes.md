# v3.1.0 — Estimate planned budget

## Summary

First **Estimate / Planned Budget** feature release within the v3 line. Adds **Planned Money** as structured line items under **Itinerary** — separate from **Expense (Actual Money)** — with CRUD CLI, shared amount/currency helpers, export **schema v6**, stats / Markdown export integration, and replicate copy behavior.

Requires prior **v3.0.0 Shared Expense** baseline (Participant + export v5). Current `trip export` writes **`schema_version: 6`**.

### Terminology

| Term | Meaning |
|---|---|
| **Estimate** | **Planned Money** — pre-trip budget line item under an Itinerary |
| **Expense** | **Actual Money** — recorded spending (unchanged semantics) |
| **Planned Budget** | **Not** a separate entity — Trip-level planned totals are **derived** from Estimate rows |
| **Planned total** | Sum of `Estimate.amount` grouped by currency (`trip stats`) |
| **Actual total** | Sum of `Expense.amount` grouped by currency (existing) |
| **Difference** | Planned vs Actual delta — **not implemented** in v3.1.0 |

## Added

### Database

New table `estimates`:

| Column | Notes |
|---|---|
| `id` | INTEGER PRIMARY KEY |
| `itinerary_id` | NOT NULL — parent Itinerary only (no Trip / Day direct link) |
| `title` | TEXT NULL |
| `amount` | INTEGER NOT NULL — smallest currency unit |
| `currency` | TEXT NOT NULL |
| `note` | TEXT NULL |
| `sort_order` | INTEGER NOT NULL DEFAULT 0 |
| `created_at` / `updated_at` | timestamps |

Index: `idx_estimates_itinerary`.

Existing DBs: `migrate_estimates` on `open_db` — idempotent table creation.

Cascade on delete:

- Itinerary delete → estimates for that itinerary removed
- Trip delete → estimates for all itineraries on the trip removed
- `db reset` clears estimates

### Shared money helpers

`src/money.rs` centralizes amount / currency parsing and validation used by **Expense** and **Estimate** (minimal currency unit INTEGER, ISO 4217-style codes).

### CLI

New commands:

```bash
estimate add --itinerary 12 --amount 14000 --currency JPY
estimate add --itinerary 12 --amount 14000 --currency JPY --title "ホテル朝食"
estimate list --trip 1
estimate list --itinerary 12
estimate show 3
estimate update 3 --amount 15000
estimate update 3 --clear-title --clear-note
estimate delete 3
```

Rules (aligned with Expense):

- `amount` + `currency` required on add
- `update` requires at least one field; `--clear-title` / `--clear-note` supported
- `--currency` without `--amount` on update → error
- JSON output via `--json` on list / show where applicable

### Export / import (schema v6)

Current `trip export` writes **`schema_version: 6`**. Nested under each itinerary:

```json
{
  "schema_version": 6,
  "days": [
    {
      "day_number": 1,
      "itineraries": [
        {
          "title": "ホテル朝食",
          "estimates": [
            {
              "amount": 14000,
              "currency": "JPY",
              "title": "朝食ビュッフェ",
              "note": null,
              "sort_order": 0
            }
          ],
          "expenses": []
        }
      ]
    }
  ]
}
```

- **v5 import** remains supported — omitted `estimates[]` → no planned rows
- **v4 / v3 import** remain supported
- `validate-export`: v6 nested `estimates[]` rules (amount / currency required)
- v5 files skip v6-only estimate checks

### Diff

`trip diff` reports Estimate changes when **both** exports are schema **v6+**:

- added / removed
- modified: `amount`, `currency`, `title`, `note`, `sort_order`

v5 vs v6 comparison does not diff estimates (v5 exports have no `estimates[]`).

### Trip stats

`trip stats` and `trip stats --json` include **Planned total** (Estimate sums by currency) alongside existing Actual totals. Multi-currency trips show separate planned lines per currency.

### Markdown export

`trip export-md` renders planned cost tables under itineraries when estimates exist. Omits the section when an itinerary has zero estimates.

### Itinerary replicate

`itinerary replicate` copies Estimate rows to newly created target itineraries via `copy_estimates_for_itinerary` (same title / amount / currency / note / sort_order; new IDs and timestamps). Dry-run does not create estimates.

`--without-estimates` is **not** provided in v3.1.0 — estimates copy by default.

## Migration / compatibility

| Topic | Notes |
|---|---|
| **Existing `caglla.db`** | Opening the DB runs `migrate_estimates` — no manual step |
| **Existing trips** | No estimates until you add them |
| **Export v5 files** | Import still works; `estimates[]` omitted → empty planned data |
| **Export v6** | Use v3.1.0+ CLI to export/import with nested `estimates[]` |
| **Downgrade** | v3.0.0 CLI cannot read `estimates` table; export v6 JSON before downgrade |

## Upgrade

1. Replace binary with v3.1.0 build (or `cargo build --release` from tag `v3.1.0`).
2. Run any command — DB migration applies automatically.
3. Add estimates with `estimate add --itinerary <id> --amount <n> --currency <code>`.
4. Re-export trips to obtain schema v6 JSON with nested `estimates[]`.
5. Use `trip stats` for Planned totals and `trip export-md` for planned cost tables in travel book output.
