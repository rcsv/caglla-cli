## Review conclusion

```text
Expense is the Transaction Record Layer of the Travel Ledger.
It records Actual Money — monetary amounts tied to Itinerary — not budget, estimate, or settlement.
```

## Financial responsibility model

```text
Budget      = Planned Money   (future)
Expense     = Actual Money    (v1.x — Transaction Record Layer)
Settlement  = Shared Expense / split result (future v3)
```

## Travel Ledger layers (with v1.20 / v1.21)

```text
Remark         = Inline Annotation
Note           = Annotation Layer
Summary        = Abstract Layer
Travel Journal = Story Layer        (future)

Reservation    = Booking Record Layer
Expense        = Transaction Record Layer
```

## Reviewed (documented)

### What Expense is / is not

| Is | Is not |
|---|---|
| Expenditure log (amount + currency) | Budget or estimate |
| Primary data for stats / Travel Book costs | Reservation (booking info) |
| Foundation for future Settlement | FX conversion or split logic (v1.x) |

### Estimate vs actual

Pre-trip “expected cost” belongs in **Note** or future **Budget**, not Expense. v1.x Expense semantically records **actual** expenditures.

### Reservation boundary

Hotel example (booked, ¥52,000): **Reservation** holds confirmation; **Expense** holds the paid amount. Both may exist on the same Itinerary; amount canonical source is **Expense only**.

### Currency

3-letter codes, unknown codes allowed, minor-unit INTEGER amounts, no conversion in v1.x — **adequate** for v2 Participant and v3 Settlement.

### v1.5+ implementation alignment

| Layer | Verdict |
|---|---|
| **DB** (`expenses` table, Itinerary parent) | **Adequate** — no schema change needed |
| **CLI** (`expense add/list/show/update/delete`) | **Valid** — not deprecated in v1.22 |
| **Export / import v3** | **Maintained** — nested `expenses[]` |
| **export-md / trip stats** | **Maintained** — primary financial aggregates |
| **trip diff** | **Not implemented** — recorded as improvement candidate |
