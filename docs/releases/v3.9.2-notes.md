# v3.9.2 — Legacy migration test hardening

## Summary

**v3.9.2** is a **test-only** release. It adds legacy database migration smoke tests so migration-order regressions (like the v3.9.1 `days.summary` issue) are caught by `make check`. No CLI behavior, SQLite schema, or trip export schema changes.

Requires **v3.9.1** baseline. Trip export still writes **`schema_version: 8`**.

---

## Added

### Legacy migration smoke tests (`src/storage/db.rs`)

Each test builds a minimal on-disk legacy SQLite fixture, then asserts `open_db_at` and `collect_db_status` succeed (same open path as `db status --json` on an existing DB).

| Test | Legacy pattern |
|---|---|
| `legacy_smoke_days_description_renamed_before_backfill` | `days.description` present, `summary` absent — rename before day backfill |
| `legacy_smoke_no_days_table_backfills_day_id` | No `days` table; `itinerary_items.day_id` absent — backfill days then `day_id` |
| `legacy_smoke_receipts_without_trashed_at` | `receipts` without `trashed_at` — column added on open |

v3.9.1 tests for `days` without `summary` (unit + `db_cli` integration) are unchanged.
