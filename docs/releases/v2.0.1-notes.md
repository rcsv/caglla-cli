# v2.0.1 — Participant Foundation hardening patch

## Summary

**Hardening patch release** after v2.0.0 Participant Foundation. Documentation and test coverage only — **no new Rust features**, **no export schema bump** (remains **v4**).

```text
v2.0.0  Participant Foundation (feature)
v2.0.1  Foundation hardening (docs + tests)  ← this release
```

**Not included:** v3 Shared Expense, Expense FK, Settlement, Person / Traveler Profile.

---

## Added

### Documentation

- **[Foundation Hardening Review](../specifications/foundation-hardening-review.md)** — v2.0.0 後・v3 前の基盤点検（PR #27）
- **export schema v4 alignment** — [export-schema.md](../specifications/export-schema.md) and [export-import.md](../export-import.md) updated for Participant export / import / diff / `validate-export` (PR #28)
- Clarified Participant behavior in user-facing export docs:
  - top-level `participants[]` on v4 export
  - v3 import compatibility when `participants` key is omitted
  - `validate-export` rejects multiple `is_self: true` per Trip
  - `trip diff` Participant added / removed / `is_self` changed

### Tests

- **`validate-export` CLI integration** — invalid v4 JSON with two `is_self: true` participants → `valid: false` (PR #28)
- **`trip export-md` regression** — asserts `## Participants` section with self / companion table (PR #28)
