# v2.0.1 — Participant Foundation hardening patch

## Summary

**Documentation / test hardening patch release** after v2.0.0 Participant Foundation.

- **No new runtime feature**
- **No export schema bump** — remains **`schema_version: 4`**
- **Cargo version** moves `2.0.0` → **`2.0.1`** only
- **v3 Shared Expense is not included**

```text
v2.0.0  Participant Foundation (feature)
v2.0.1  Foundation hardening (docs + tests + design principles)  ← this release
```

**Not included:** v3 Shared Expense, Expense FK, Settlement, Person / Traveler Profile.

## Added

### Documentation

- **[Foundation Hardening Review](../specifications/foundation-hardening-review.md)** — post-v2.0.0 baseline review before v3 (PR #27)
- **export schema v4 alignment** — [export-schema.md](../specifications/export-schema.md) and [export-import.md](../export-import.md) for Participant export / import / diff / `validate-export` / markdown (PR #28)
- **[Planning Design Principles](../specifications/planning-design-principles.md)** — Itinerary / Checklist / Note / Reservation / Expense role separation (PR #29)

Participant export behavior clarified (PR #28):

- top-level `participants[]` on v4 export
- v3 import when `participants` key is omitted
- `validate-export` rejects multiple `is_self: true` per Trip
- `trip diff` Participant added / removed / `is_self` changed

Planning design principles clarified (PR #29):

- **Itinerary is not a task row** — travel activity flow; **no `is_done` by design**
- **Checklist** is the place for confirmation tasks — **`is_done` belongs here**
- **Notes remain flexible** — unstructured escape hatch; avoid over-structuring
- **Reservation multiplicity** — allowed; multiple per Itinerary may hint at coarse granularity (future doctor/advisor candidate)
- **Expense multiplicity** — multiple expenses under one Itinerary is natural (same activity, multiple transactions)

### Tests

- **`validate-export` CLI integration** — invalid v4 JSON with two `is_self: true` participants → `valid: false` (PR #28)
- **`trip export-md` regression** — asserts `## Participants` section with self / companion table (PR #28)
