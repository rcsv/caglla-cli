# v4.6.6 — Trip show service boundary

## Summary

**v4.6.6** introduces a minimal **service layer** for the read-only `trip show` command, keeping existing CLI human/JSON output and not-found behavior unchanged while making the use case reusable for future Tauri GUI detail screens.

Specification: [v4.6.6-trip-show-service-boundary.md](../specifications/v4.6.6-trip-show-service-boundary.md)

## What changed

- New `src/services/trip_show.rs` — `show_trip()` returns `TripShowServiceResult` (no terminal I/O)
- `trip show` CLI handler calls service, then routes to existing human/JSON output

## Compatibility

- `trip show` human output unchanged
- `trip show --json` output unchanged (serializes `Trip`)
- not found behavior unchanged (`Trip not found: <id>`)
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No other read-only commands (`day list`, `itinerary list/timeline`, etc.)
- No write command service split
- No Tauri / GUI implementation
