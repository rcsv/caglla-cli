# v1.0.1 Not found handling polish

## Summary

This patch release improves not-found error handling for Trip, Itinerary, and Checklist commands.

## Changes

- Convert missing records into clear domain errors:
  - `Trip not found: <id>`
  - `Itinerary not found: <id>`
  - `Checklist item not found: <id>`
- Hide internal SQLite message `Query returned no rows`
- Preserve existing failure behavior:
  - exit code `1`
  - stderr text error output
  - empty stdout for `--json` error cases
- Add unit and CLI integration tests for not-found behavior
