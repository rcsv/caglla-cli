# v4.6.5 — Read-only service boundary expansion

## Summary

**v4.6.5** expands the read-only **service boundary** approach from v4.6.4 (`trip stats`) to **`trip list`**, keeping existing CLI human/JSON output unchanged while making the use case reusable for future Tauri GUI entrypoints.

Specification: [v4.6.5-read-only-service-boundary-expansion.md](../specifications/v4.6.5-read-only-service-boundary-expansion.md)

## What changed

- New `src/services/trip_list.rs` — `list_trips()` returns `TripListServiceResult` (no terminal I/O)
- `trip list` CLI handler calls service, then routes to existing human/JSON output

## Compatibility

- `trip list` human output unchanged
- `trip list --json` output unchanged (serializes `Vec<Trip>`)
- Ordering unchanged (`ORDER BY id`)
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No other read-only commands (`trip show`, `day list`, `itinerary list/timeline`, etc.)
- No write command service split
- No Tauri / GUI implementation
