# v4.1.0 — Travel Book chapter structure design

## Summary

**v4.1.0** is a **documentation-only release**. It defines the **Markdown chapter structure** for Travel Book: chapter order, entity mapping per chapter, empty-chapter rules, and a mandatory **Colophon** at the end. Builds on v4.0.0 concept design. No CLI behavior, SQLite schema, or trip export schema changes.

Requires **v4.0.0** baseline. Trip export still writes **`schema_version: 8`**.

Design: [v4.1.0-travel-book-chapter-structure-design.md](../specifications/v4.1.0-travel-book-chapter-structure-design.md)


## Added

### Chapter structure design document

[v4.1.0-travel-book-chapter-structure-design.md](../specifications/v4.1.0-travel-book-chapter-structure-design.md) — canonical chapter layout:

1. **Cover** — always output (trip name, dates, participant count)
2. **Trip overview** — summary, participants, stats; omit when empty
3. **Highlights** — omit when empty (no auto-extraction yet)
4. **Daily schedule** — always output; Day / Itinerary sequence-first
5. **Reservations** — omit when no reservations
6. **Checklist** — omit when empty
7. **Planned cost** — Estimate-focused; omit when no estimates
8. **Notes** — Note entities; omit when empty
9. **Colophon** — always output at end (generator credit, version, timestamp, trip name/dates; **no DB path** in normal profile)
