# v3.9.0 — Config and DB path foundation

## Summary

**v3.9.0** adds explicit **database path resolution** for the Caglla CLI. You can select which SQLite file to use via `--db`, `CAGLLA_DB`, or a local `./caglla.toml` config file. Default behavior remains CWD + `caglla.db`.

Requires **v3.8.0** baseline. Trip export still writes **`schema_version: 8`**.

Design: [v3.9.0-config-and-db-path-foundation-implementation-plan.md](../specifications/v3.9.0-config-and-db-path-foundation-implementation-plan.md)

---

## Added

### Global `--db <path>`

```bash
caglla --db ./sample.db db path
caglla --db ./sample.db trip list
caglla trip list --db ./sample.db
```

### Environment variable `CAGLLA_DB`

```bash
CAGLLA_DB=./env.db caglla db path
```

### Local config `./caglla.toml`

```toml
[database]
path = "./caglla.db"
```

Relative paths in `path` resolve against the config file’s directory. Phase 1 reads **only** `./caglla.toml` in the current working directory.

### Resolution priority

```text
1. --db <path>
2. CAGLLA_DB
3. ./caglla.toml [database].path
4. ./caglla.db (default)
```

### `db status` enhancements

- Human output includes **Path source** (`cli` / `env` / `config` / `default`).
- JSON schema version **2** adds `path_source` and optional `config_path` (additive; trip export schema unchanged).

### `db reset` output

Shows the resolved database path:

```text
【開発用】データベースを初期化しました
  DB: /abs/path/to/selected.db
```

## Unchanged

| Item | Notes |
|---|---|
| SQLite schema / migrations | No changes |
| `trip export` JSON schema | Still **v8** |
| `db path` / `db status` (missing DB) | Still do **not** create the database file |
| Receipt / Expense / Estimate semantics | Unchanged |

## Not in v3.9.0

```text
db use
caglla.toml auto-generation
user-global config
parent directory config search
profile switching
remote DB / cloud sync
```

## Upgrade notes

- `db status --json` now reports `schema_version: 2` with `path_source` and `config_path`.
- Without `--db`, `CAGLLA_DB`, or `caglla.toml`, behavior matches previous releases (`./caglla.db` in CWD).
