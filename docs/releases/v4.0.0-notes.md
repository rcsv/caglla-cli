# v4.0.0 — Travel Book concept design

## Summary

**v4.0.0** is a **documentation-only release** — the product-major milestone for **Travel Book**: a pre-trip shareable itinerary document derived from Caglla data. It positions existing `trip export-md` as **Travel Book Generator v0** and defines purpose, information boundaries, and output scope. No CLI behavior, SQLite schema, or trip export schema changes.

Requires **v3.11.0** baseline. Trip export still writes **`schema_version: 8`**.

Design: [v4.0.0-travel-book-concept-design.md](../specifications/v4.0.0-travel-book-concept-design.md)

Design series:

```text

```

## Added

### Travel Book concept design document

[v4.0.0-travel-book-concept-design.md](../specifications/v4.0.0-travel-book-concept-design.md) — canonical v4 design:

- **Purpose** — pre-trip shareable “しおり” for companions, not a DB dump
- **`trip export-md`** — Travel Book Generator v0; v4 evolves layout and boundaries, not a parallel dump format
- **Information scope** — Trip / Day / Itinerary / Summary / Reservation / Checklist / Estimate primary; Receipt and pending sums excluded
- **Money model** — Planned (Estimate) centered; Expense / Actual subdued for pre-trip output
- **Travel Journal** — separate v5+ concept (photos, attachments, post-trip record)
- **Output modes** — Markdown first; PDF deferred until structure stabilizes
- **Profiles** — public / private / debug noted for future; not implemented in v4.0.0

## Clarified

| Topic | Clarification |
|---|---|
| **Travel Book vs Travel Journal** | Book = before/during trip sharing; Journal = after-trip record (v5+) |
| **Receipt in Book** | Not shown — inbox candidates, not Actual |
| **Reservation** | v1.18.0 entity reused; v4 focus is display integration in the book |
| **Summary** | Abstract — short trip/day overview, not a journal entry |
| **CLI in v4.0.0** | No changes; `export-md` evolution is a later implementation milestone |
