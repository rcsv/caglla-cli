# v3.8.0 — Roadmap realignment after Receipt Inbox

## Summary

**v3.8.0** is a **documentation-only release**. It realigns the product major-version roadmap after v3.7.1 completed the Receipt Inbox workflow (assign / trash / Okinawa sample / export fix). No CLI behavior, DB schema, or export/import schema changes.

Requires **v3.7.1** baseline. Current trip export still writes **`schema_version: 8`**.

Design series:

```text
v3.7.1 Receipt Inbox complete        → v3.7.1-notes.md
Roadmap realignment                  → v3.8.0-roadmap-realignment-after-receipt-inbox.md
Release v3.8.0                       → this release
```

## Clarified

| Topic | Clarification |
|---|---|
| **v4 Reservation** | **Not a future major milestone** — fulfilled in v1.18.0. Future reservation *enhancements* belong under Travel Book or patch-level design. |
| **Receipt Inbox** | v3.5–v3.7.1 scope is **closed** for core workflow; purge / images / OCR remain deferred. |
| **Travel Book vs export-md** | `trip export-md` is **Travel Book Generator v0** today; v4 targets richer pre-trip shared output (MD/PDF, layout). |
| **Travel Journal** | Remains **v5** and **blocked** until Evidence / Attachment common layer is designed. |
| **Next work** | Prefer **one** design topic: DB path, Travel Book concept, or doctor-advisor utilization. |
