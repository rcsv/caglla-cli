| Item | Status |
|---|---|

| 観点 | 判定 |

## Background

目標形:

```text
CLI / Tauri
  → service (use case, structured result, no I/O)
  → repository / domain
```

`checklist list` は Trip 単位のチェックリスト一覧。write 系（add / check / auto-generate 等）には触れず、既存挙動を保ったまま service boundary のみ導入する。

---

## Target command

- `checklist list <trip_id>`
- `--json` フラグあり（`Vec<ChecklistItem>` をそのまま serialize）

`--day` / `--category` フィルタは存在しない（従来どおり Trip 単位のみ）。

---

## Current flow before split

```text
main.rs (ChecklistAction::List)
  → checklist::list_checklist_items(conn, trip_id)
  ├─ --json  → print_json(&items)
  └─ human   → println!("旅行 ID {trip_id} のチェックリスト:")
             → print_checklist_list(&items)
```

---

## Service boundary introduced

新規:

```text
src/services/checklist_list.rs
```

API:

```rust
pub struct ChecklistListServiceResult {
    pub items: Vec<ChecklistItem>,
}

pub fn list_checklist(conn: &Connection, trip_id: i64) -> Result<ChecklistListServiceResult>
```

service は `checklist::list_checklist_items` を呼び、**structured result のみ**返す。

human 見出し（`旅行 ID {trip_id} のチェックリスト:`）は CLI 層に残す。

---

## CLI compatibility

- human: 見出し `println!` + `print_checklist_list(&result.items)`
- JSON: `result.items` をそのまま serialize

---

## JSON compatibility

- `--json` 時は `Vec<ChecklistItem>` を serialize（shape 不変、wrapper なし）

---

## Error compatibility

- Trip 不存在: `Trip not found: <id>`（`list_checklist_items` 内の `get_trip` 経由）

---

## Ordering compatibility

```text
ORDER BY is_done ASC, sort_order ASC, id ASC
```

未完了項目が先、完了項目が後。service は順序を変えない。

---

## checked / unchecked compatibility

- `is_done` フィールドは `ChecklistItem` に含まれる
- human 表示の `[x]` / `[ ]` マークは `print_checklist_list` 内（変更なし）

---

## Test coverage

  - ordering + checked state 維持

## What changed

- `src/services/checklist_list.rs` 追加
- `src/services/mod.rs` に `checklist_list` module 追加
- `main.rs` の `ChecklistAction::List` が service 経由に

---

## What did not change

- `checklist show` / add / update / delete / check / uncheck（対象外）
- `checklist auto-generate`（対象外）
- `expense list` / `reservation list` / `note list`（対象外）
- write command / migration / FK（変更なし）

---

## Lessons learned

- Trip 単位のみの list は `itinerary list` と同様に薄い service で十分
- JSON に wrapper がない command は `result.items` を直接 serialize
- human 見出しは CLI 層に残す（timeline / checklist list パターン）
