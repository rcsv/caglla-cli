## 1. 背景

Caglla.Travel CLI は v1〜v3 系で、旅行計画・同行者・費用・見積・実績・Receipt Inbox・DB/config 基盤を整備してきた。

これまでの主な到達点:

```text
v1 Planning Foundation
  Trip / Day / Itinerary / Checklist / Note / Summary / Reservation / Expense

v2 Participant Foundation
  Participant

v3 Shared Expense / Planned vs Actual / Receipt Inbox / DB config
  Paid By / Beneficiaries
  Estimate
  Planned vs Actual
  Receipt Inbox
  --db / CAGLLA_DB / caglla.toml / db use
```

一方で、ユーザー視点では、旅行前に最も欲しいものの一つは **「家族や同行者に共有できる、見やすい旅のしおり」** である。

現行の `trip export-md` は、その原型として機能している。
v4 Travel Book では、`trip export-md` を **Travel Book Generator v0** と位置づけ、旅行前共有用ドキュメントとしての責務・構成・将来拡張を整理する。

---

## 2. Travel Book の目的

Travel Book の目的は、Caglla に蓄積された旅行データから、旅行前に共有できる **読み物としての旅程ドキュメント** を生成することである。

```text
Travel Book = 旅行前に共有するための「しおり」
```

対象読者は、CLI 利用者本人ではなく、同行者を含む。

例:

```text
- 家族旅行の同行者
- 高齢の親
- 子ども
- 旅行前に予定をざっと確認したい人
- 当日スマホで見る人
- 紙に印刷して持ちたい人
```

Travel Book は、単なる DB dump ではない。
旅行中に見返しやすいように、重要な情報を選び、順序立て、読みやすく整える出力物である。

---

## 3. 現行 `trip export-md` の位置づけ

現行の `trip export-md` は、Travel Book の原型である。

```text
trip export-md = Travel Book Generator v0
```

ただし、現時点では以下の性格が強い。

| 現行 export-md | 説明 |
|---|---|
| CLI 由来の構造出力 | DB 内容を比較的素直に Markdown 化する |
| 開発・確認向け | データ確認・設計検証にも使われる |
| 表示階層は限定的 | 「しおり」としての読みやすさはまだ弱い |
| 旅行前共有専用ではない | 計画・実績・内部情報が混ざりやすい |

v4 Travel Book では、`trip export-md` を単に増築するのではなく、**旅行前共有ドキュメントとしての視点**を導入する。

---

## 4. Travel Book が扱う情報

Travel Book は、既存エンティティを再利用する。
v4 concept design 時点では、新しい TravelBook エンティティを必須とはしない。

### 4.1 主に表示する情報

| 情報 | Travel Book での扱い |
|---|---|
| Trip | 表紙・旅行全体概要 |
| Trip Summary | 旅行全体の短い説明 |
| Day | 日別セクション |
| Day Summary | その日の見どころ・注意点 |
| Itinerary | 旅程本体 |
| Reservation | 予約・確認番号・時間・場所 |
| Checklist | 旅行前準備リスト |
| Estimate | 予定費用の参考表示 |
| Expense | 原則として旅行前しおりでは控えめ |
| Participant | 同行者情報。ただし個人情報の扱いに注意 |
| Note | 補足説明として利用可能 |

### 4.2 原則として表示しない、または慎重に扱う情報

| 情報 | 理由 |
|---|---|
| Receipt | 未整理支払い候補であり、旅行前共有には不要 |
| Pending Receipt sum | Actual ではなく、Travel Book には混ぜない |
| Internal ID | 読み物としては不要。必要なら debug/profile 表示 |
| DB path / config 情報 | しおり読者には不要 |
| migration / schema 情報 | しおり読者には不要 |
| 個人情報 | Participant / Reservation に含まれる可能性があるため注意 |

## 5. Planned / Actual / Receipt の境界

v3 系で整理した money model は維持する。

```text
Estimate = Planned
Expense  = Actual
Receipt  = 未整理支払い候補。Actual ではない
```

Travel Book は旅行前共有ドキュメントであるため、基本的には **Planned 情報を中心**に扱う。

| 種類 | Travel Book での扱い |
|---|---|
| Estimate | 予定費用として表示候補 |
| Expense | 旅行前時点では通常表示しない。旅行後出力では別 profile を検討 |
| Receipt | 表示しない |
| Pending Receipt sum | 表示しない |
| Planned vs Actual Difference | Travel Book v4 初期では非対象 |

将来、旅行後向けの出力が必要になった場合は、Travel Journal または post-trip report として分離する。

---

## 6. Travel Book と Travel Journal の違い

Travel Book と Travel Journal は分ける。

```text
Travel Book    = 旅行前に共有するしおり
Travel Journal = 旅行後に残す記録
```

| 観点 | Travel Book | Travel Journal |
|---|---|---|
| 時点 | 旅行前・旅行中 | 旅行後 |
| 主目的 | 予定共有・当日確認 | 記録・思い出 |
| 主な情報 | Itinerary / Summary / Reservation / Checklist | Photo / Attachment / Note / 実績 |
| Receipt | 原則不要 | 将来 Evidence として扱う可能性 |
| Photo / Attachment | 非対象 | v5 以降候補 |
| OCR | 非対象 | Evidence 設計後 |

v4 では Travel Journal 実装に入らない。
Photo / Attachment / Evidence / OCR は引き続き defer する。

## 7. Travel Book の出力モード

v4 concept design 時点では、まず Markdown を中心に考える。

### 7.1 Markdown

```text
trip export-md
```

現行の延長として最も自然。
GitHub / iPhone / Mac / 印刷前プレビューなどで扱いやすい。

### 7.2 PDF

PDF は Travel Book の自然な到達点だが、v4.0.0 concept design では実装しない。

PDF は以下の論点を持つ。

```text
- フォント
- 改ページ
- 表組み
- 画像
- 日本語レイアウト
- 印刷余白
- GitHub Actions 上での生成
```

そのため、PDF は v4 初期実装ではなく、Markdown 構造が安定してから扱う。

### 7.3 HTML

将来の Web / GUI と相性は良いが、CLI v4 の初期スコープでは優先しない。
必要に応じて Markdown → HTML 変換で代替できる。

## 8. Travel Book の基本構成案

Travel Book は、旅行前に読む人が迷わない順序にする。

```text
1. 表紙
2. 旅行概要
3. 重要ポイント
4. 日別旅程
5. 予約情報
6. 持ち物・準備
7. 費用の目安
8. 補足メモ
```

### 8.1 表紙

表示候補:

```text
- Trip name
- Date range
- Destination summary
- Participant count
- Generated at
```

### 8.2 旅行概要

Trip Summary を中心に表示する。

```text
Summary = Abstract
```

長い旅行記ではなく、旅行全体をざっと理解するための短い説明として扱う。

### 8.3 重要ポイント

自動生成できる可能性がある情報:

```text
- 固定時刻のある予定
- 予約がある予定
- 移動・チェックイン・チェックアウト
- 朝早い出発
- 事前準備が必要なもの
```

ただし v4.0.0 concept design では、重要ポイントの自動抽出は実装しない。
将来の doctor / advisor と連動する可能性を残す。

### 8.4 日別旅程

Day ごとに以下を表示する。

```text
Day 1 — YYYY-MM-DD
Summary
Itinerary list
Reservation highlights
Notes
```

Itinerary は sequence-first ordering を維持する。

```text
sort_order を主、start_time は任意ラベル
```

### 8.5 予約情報

Reservation は v1.18.0 で実装済みであり、Travel Book では表示統合対象とする。

表示候補:

```text
- reservation_type
- provider_name
- confirmation_code
- start_at / end_at
- contact
- note
```

ただし、個人情報や確認番号の扱いには注意する。
将来、public / private 表示 profile を検討してもよい。

### 8.6 持ち物・準備

Checklist を Travel Book に表示する。
旅行前共有では有用性が高い。

表示候補:

```text
- 未完了 / 完了
- category
- title
```

ただし、現時点では Checklist のカテゴリ構造が限定的であれば、既存表示を活かす。

### 8.7 費用の目安

Travel Book は旅行前共有用なので、Estimate を中心に表示する。

```text
Planned total
Itinerary-level planned cost
```

Expense / Actual は初期 Travel Book では控えめにする。
旅行後 report とは分ける。

### 8.8 補足メモ

Note は Annotation として扱う。

```text
Note = 補足・注釈
Summary = 要約
Journal = 旅行記
```

この境界を維持する。

## 9. 出力 profile の考え方

Travel Book では将来、出力 profile を分ける可能性がある。

例:

```text
public      同行者共有向け。個人情報を抑制
private     自分用。予約番号や詳細メモも含める
debug       開発・検証用。ID や内部情報を含める
```

v4.0.0 では profile 実装はしない。
ただし、今後の設計で private 情報の扱いが問題になるため、Concept Design に論点として残す。

初期実装では、既存 `trip export-md` の互換を重視し、過度な出し分けは避ける。

---

## 10. CLI 方向性

v4 初期では、既存の `trip export-md` を拡張する方向を基本とする。

候補:

```bash
caglla trip export-md <trip-id> --output trip.md
```

将来候補:

```bash
caglla trip export-book <trip-id> --output trip.md
caglla trip export-book <trip-id> --format markdown
caglla trip export-book <trip-id> --profile public
```

ただし、新コマンドを増やすか、既存 `export-md` を進化させるかは Implementation Plan で判断する。

v4.0.0 concept design では CLI 変更は行わない。

---

## 11. Non-goals

v4.0.0 concept design では以下を行わない。

```text
- Travel Book 実装
- PDF export 実装
- HTML export 実装
- template engine 導入
- 新 DB entity 追加
- DB schema 変更
- trip export schema 変更
- Attachment / Photo / Evidence 設計
- OCR / automatic receipt parsing
- Travel Journal 実装
- Balance / Settlement
- Cloud / Identity / Platform
```
