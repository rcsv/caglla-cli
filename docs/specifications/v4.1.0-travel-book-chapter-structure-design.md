## 1. 目的

v4.0.0 で Travel Book の概念・情報境界を定義した。v4.1.0 では次を決める。

```text
- しおりを「どの順番で読むか」（章順）
- 各章にどの既存 entity を載せるか
- データが無い章を出すか（空章ポリシー）
- 奥付（生成情報）を末尾に常設するか
```

これにより、v4.2 以降の `trip export-md` リファクタは **章単位の実装タスク** に分解できる。

---

## 2. 章一覧（正本）

Travel Book Markdown は、先頭から次の順で構成する。

| # | 章 ID | 見出し（英） | 見出し（日） | 空章 |
|---|---|---|---|---|
| 1 | `cover` | Cover | 表紙 | **常に出力** |
| 2 | `trip_overview` | Trip overview | 旅行概要 | 条件付き（§4.2） |
| 3 | `highlights` | Highlights | 重要ポイント | **省略可** |
| 4 | `daily_schedule` | Daily schedule | 日別旅程 | **常に出力**（Trip 有効時） |
| 5 | `reservations` | Reservations | 予約情報 | **省略可** |
| 6 | `checklist` | Checklist | 持ち物・準備 | **省略可** |
| 7 | `planned_cost` | Planned cost | 費用の目安 | **省略可** |
| 8 | `notes` | Notes | 補足メモ | **省略可** |
| 9 | `colophon` | Colophon | 奥付 | **常に出力** |

```text
Cover → Trip overview → Highlights → Daily schedule → Reservations
     → Checklist → Planned cost → Notes → Colophon
```

v4.0.0 §8 の構成案と対応:

| v4.0.0 案 | v4.1.0 章 |
|---|---|
| 表紙 | Cover |
| 旅行概要 | Trip overview |
| 重要ポイント | Highlights |
| 日別旅程 | Daily schedule |
| 予約情報 | Reservations |
| 持ち物・準備 | Checklist |
| 費用の目安 | Planned cost |
| 補足メモ | Notes |
| （未定義） | **Colophon**（v4.1.0 で追加） |

## 3. 章と entity の対応

新エンティティは導入しない。既存モデルのみを章に割り当てる。

### 3.1 Cover（表紙）

| ソース | フィールド / 意味 | 表示 |
|---|---|---|
| Trip | `name` | 旅行名（主タイトル） |
| Trip | `start_date` / `end_date` | 旅行期間 |
| Trip | （将来）destination 要約 | 任意。現行 Trip に専用フィールドなし — Summary から要約文を流用しない（概要章と分離） |
| Participant | 件数 | 同行者数（`is_self` を除いた companion 数 + self の有無は Overview へ） |

**含めない:** Internal ID、DB path、Receipt、Expense、schema 情報。

### 3.2 Trip overview（旅行概要）

| ソース | フィールド / 意味 | 表示 |
|---|---|---|
| Trip | `summary` | 旅行全体の短い説明（**Summary = Abstract**） |
| Participant | `name`, `is_self` | 同行者一覧（個人情報 — 将来 public profile で抑制可） |
| Trip stats（派生） | 日数、Itinerary 件数、参加者数 | 数値サマリー（現行 `export-md` の Overview に相当） |

**含めない:** Planned vs Actual 差分、Receipt、Pending sum、Expense 明細。

### 3.3 Highlights（重要ポイント）

v4.0.0 では自動抽出は未実装。v4.1.0 は **章の器のみ** 定義する。

将来の候補ソース（v4.2 以降で検討）:

| 候補 | 根拠 |
|---|---|
| Itinerary + `start_time` 固定 | 時刻付き予定 |
| Reservation 紐づき Itinerary | 予約あり行動 |
| doctor / advisor issue | 注意喚起 |

**v4.1.0 設計時点:** 抽出ロジックなし → **章全体を省略**（空章を出さない）。

### 3.4 Daily schedule（日別旅程）

| ソース | フィールド / 意味 | 表示 |
|---|---|---|
| Day | `day_number`, 派生日付 | 日見出し |
| Day | `summary` | その日の概要 |
| Itinerary | `title`, `start_time`, `category`, `location`, `duration_minutes`, `travel_minutes`, `note` | 旅程本体 |
| Itinerary 配下 Estimate | 予定費用 | **日別章内では省略** — Planned cost 章へ集約（v4.2 実装方針） |
| Itinerary 配下 Expense | 実績 | **旅行前しおりでは省略**（v4.0.0 維持） |
| Note（Day / Itinerary スコープ） | `body` | 日別旅程内の補足として **短く** 載せるか、Notes 章へ集約 — **v4.2 実装で二选一**。v4.1.0 では Notes 章への集約を推奨 |
| Reservation | — | 日別章では **ハイライト行のみ** 可（詳細は Reservations 章）。v4.2 実装判断 |

**順序:** sequence-first（`sort_order` 主、`start_time` は表示ラベル）。

### 3.5 Reservations（予約情報）

| ソース | フィールド / 意味 | 表示 |
|---|---|---|
| Reservation | `reservation_type`, `provider_name`, `confirmation_code`, `start_at`, `end_at`, `reservation_site_url`, `remark` | 予約ブロック |
| 文脈 | 紐づく Day / Itinerary | 見出し補助（現行 export-md と同様） |

**含めない:** Receipt、Expense。

**個人情報:** 確認番号・連絡先は将来 public profile でマスク可（v4.0.0 §9）。

### 3.6 Checklist（持ち物・準備）

| ソース | フィールド / 意味 | 表示 |
|---|---|---|
| ChecklistItem | `title`, `is_done`, `category`（あれば） | チェックリスト |

### 3.7 Planned cost（費用の目安）

| ソース | フィールド / 意味 | 表示 |
|---|---|---|
| Estimate | Itinerary 配下 + Trip 集計 | 通貨別 Planned total |
| Trip stats（派生） | `estimate_count`, planned totals | 章頭サマリー |

**含めない:** Expense（Actual）、Planned vs Actual Difference、Receipt。

### 3.8 Notes（補足メモ）

| ソース | フィールド / 意味 | 表示 |
|---|---|---|
| Note | Trip / Day / Itinerary スコープの `body` | Annotation として一覧またはグループ表示 |

**境界:**

```text
Summary = 要約（Trip overview / Day summary に載せる）
Note    = 補足・注釈（本章）
```

Itinerary の `note` フィールド（単一行メモ）と Note entity は別。両方ある場合は Daily schedule と Notes の分担を v4.2 で明文化する。

### 3.9 Colophon（奥付）

Travel Book **末尾** に置く生成情報セクション。**データ有無にかかわらず常に出力**する。

| 項目 | 内容 | 備考 |
|---|---|---|
| Generator credit | `Generated by Caglla.Travel CLI` | 固定文言 |
| Generator name | `Travel Book Generator v0` | v4.2 実装まで現行名を維持 |
| Caglla version | Cargo package version | ビルド時の CLI バージョン |
| Generated at | ISO 8601 タイムスタンプ | 生成日時 |
| Trip name | Trip.name | 再掲 |
| Trip date range | start — end | 再掲 |

**通常 profile では出さない:**

```text
- source DB path（ローカル環境情報。共有しおりに不要）
- config path / CAGLLA_DB / --db
- migration / schema_version
- trip internal id（debug profile のみ将来候補）
```

将来 `debug` profile を導入する場合のみ、DB path 等を奥付または付録に載せられる。

## 4. 空章ポリシー

### 4.1 常に出力する章

| 章 | 理由 |
|---|---|
| Cover | しおりの入口。Trip 名・期間は常に意味がある |
| Daily schedule | 旅行の本体。Day が 0 件の Trip は通常存在しないが、空でも見出しのみ出す（「旅程未登録」等は v4.2 実装） |
| Colophon | 生成物の出所・バージョン追跡。共有・印刷時の信頼性 |

### 4.2 条件付きで出力する章

**Trip overview**

```text
次のいずれかがあれば章を出す:
  - Trip.summary が非空
  - Participant が 1 件以上
  - 表示対象の stats 行が 1 つ以上

すべて空なら章ごと省略（Cover に Trip 名・期間は既にある）
```

### 4.3 省略可能な章（データなし → 章ごと出さない）

| 章 | 省略条件 |
|---|---|
| Highlights | 抽出済みハイライトが 0 件（v4.1 時点は常に省略） |
| Reservations | Reservation が 0 件 |
| Checklist | ChecklistItem が 0 件 |
| Planned cost | Estimate が 0 件（Planned total なし） |
| Notes | 対象 Note が 0 件 |

**原則:** 空見出し（`## Reservations` のみで本文なし）は **出さない**。読者に「欠けた章」と誤解させない。

### 4.4 空章と奥付

Colophon は **常に出力**。他章がすべて省略されても、Cover + Daily schedule（最小）+ Colophon で完結するしおりになりうる。

## 5. 現行 `trip export-md` との差分

現行 Generator v0（`src/io/markdown.rs`）は章構造が未整理である。v4.2 で本設計に寄せる。

| 本設計の章 | 現行 export-md のおおよその位置 | ギャップ |
|---|---|---|
| Cover | `# Trip name` + summary 混在 + 日付 | 表紙と概要が未分離 |
| Trip overview | `## Overview` + `## Participants` | 章名・順序が未統一 |
| Highlights | なし | 章なし |
| Daily schedule | Day 見出し + Itinerary 直列 | Expense / 差分が混在 |
| Reservations | `## Reservations`（Overview 後） | 順序が設計と異なる（設計は Daily の後） |
| Checklist | 末尾付近 | 順序要調整 |
| Planned cost | Overview / Itinerary 内分散 | 独立章なし |
| Notes | Trip note 等が散在 | 集約章なし |
| Colophon | なし | **未実装** |

**v4.1.0 ではコード変更しない。** 上表は v4.2 Implementation Plan の入力とする。

## 6. Markdown 見出しレベル（推奨）

実装時のデフォルト案（v4.2 参照）:

| 章 | 推奨見出し |
|---|---|
| Cover | `# {Trip name}`（表紙タイトル） |
| Trip overview | `## Trip overview` |
| Highlights | `## Highlights` |
| Daily schedule | `## Daily schedule` → `### Day N — YYYY-MM-DD` → Itinerary `####` |
| Reservations | `## Reservations` |
| Checklist | `## Checklist` |
| Planned cost | `## Planned cost` |
| Notes | `## Notes` |
| Colophon | `## Colophon` |

言語（英 / 日）は v4.2 で統一ルールを決める。v4.1.0 では章 ID を正本とし、表示ラベルは実装時に確定可。

## 7. 明示的に Travel Book に載せないもの

v4.0.0 を維持:

```text
Receipt / Pending Receipt sum
Expense（旅行前しおりの主役にしない）
Planned vs Actual Difference
Internal ID（通常 profile）
DB path / config（奥付にも通常は出さない）
```
