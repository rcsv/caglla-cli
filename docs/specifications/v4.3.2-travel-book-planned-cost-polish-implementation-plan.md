# v4.3.2 — Travel Book planned cost polish (implementation)

Implements the **Planned cost Note column** polish identified in the v4.3.1 post-release review.

| Item | Status |
|---|---|
| Release | v4.3.2 |
| Scope | `trip export-md` Planned cost table formatting only |
| Validation fixture | [Okinawa Sesoko 2026 sample](../../samples/okinawa_sesoko_2026/README.md) |

## Goal

When all Estimate `note` values are empty within a single Planned cost table, omit the `Note` column to improve readability.

## Rule

- Per **one Planned cost table** (the table under a single `### Day N / ...` group):
  - If **all** notes are empty (empty string or whitespace), omit the `Note` column.
  - If **any** note is non-empty, keep the `Note` column (existing behavior).
- Planned totals / estimate counts are unchanged.
- No changes to Estimate / Expense / Receipt responsibilities, DB schema, or export JSON schema.

## Code

- `src/io/markdown.rs`
  - `format_planned_cost_chapter` — decide column set per table group

## Tests

- `src/io/markdown.rs` unit tests
  - Omits note column when all empty
  - Keeps note column when any present
- `tests/export_md_cli.rs` — adjusted assertion for null-title/note estimate row
