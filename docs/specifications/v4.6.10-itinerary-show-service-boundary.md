# Itinerary show service boundary

v4.6.4〜v4.6.9 の read-only service boundary に続き、**`itinerary show` 単独**で service 層を導入する。

| Item | Status |
|---|---|
| Target | v4.6.10 — itinerary show service boundary |
| Target command | **`itinerary show` のみ** |
| Scope | service 層導入（structured result）、CLI 出力互換維持 |

---

## Status

| 観点 | 判定 |
|---|---|
| service layer | **`src/services/itinerary_show.rs`** 追加 |
| target command | `itinerary show` が service 経由 |
| terminal I/O | service 内に `println!` / `eprintln!` なし |
| CLI human output | **不変**（詳細 + 予約一覧セクション） |
| CLI JSON output | **不変**（`ItineraryItem` 単体 serialize） |
| Itinerary not found | **不変**（`Itinerary not found: <id>`） |
| DB schema / migration | **変更なし** |

---

## Background

目標形:

```text
CLI command
  → service/usecase
  → repository
  → SQLite

Tauri command
  → same service/usecase
  → repository
  → SQLite
```

`itinerary show` は GUI の Itinerary detail 表示に直結する。v4.6.10 では詳細表示仕様改善は行わず、既存挙動を保ったまま service boundary のみ導入する。

---

## Target command

- `itinerary show <id>`
- `--json` フラグあり（`ItineraryItem` をそのまま出力）

---

## Current flow before split

```text
main.rs (ItineraryAction::Show)
  → itinerary::get_itinerary_item(conn, id)
  ├─ --json  → output::json::print_json(&item)
  └─ human   → print_itinerary_detail(&item)
             → reservation::list_reservations_for_itinerary(conn, id)  [human only]
```

DB 取得と terminal 出力が handler に混在していた。

---

## Service boundary introduced

新規:

```text
src/services/itinerary_show.rs
```

API:

```rust
pub struct ItineraryShowServiceResult {
    pub item: ItineraryItem,
}

pub fn show_itinerary(conn: &Connection, id: i64) -> Result<ItineraryShowServiceResult>
```

service は `itinerary::get_itinerary_item` を呼び、**structured result のみ**返す。

human 表示の予約一覧は CLI 層が従来どおり `list_reservations_for_itinerary` で取得する（JSON には含めない）。

---

## CLI compatibility

- human 出力は従来どおり:
  - `print_itinerary_detail(&result.item)`
  - 予約がある場合のみ `Reservations (N):` セクション（handler の `println!`）
- JSON 出力は `result.item` を serialize（shape 不変）

---

## JSON compatibility

- `--json` 時は `ItineraryItem` 単体を serialize（変更なし）
- wrapper や追加フィールドなし

---

## Error compatibility

- Itinerary 不存在: `Itinerary not found: <id>`（`get_itinerary_item` 経由）

---

## Test coverage

- service test:
  - 既存 itinerary の取得
  - Itinerary not found message 維持
- 既存 `not_found_cli` / `reservation_cli` テストを回帰として維持

---

## What changed

- `src/services/itinerary_show.rs` 追加
- `src/services/mod.rs` に `itinerary_show` module 追加
- `main.rs` の `ItineraryAction::Show` が service 経由に

---

## What did not change

- `itinerary list` / `itinerary timeline`（対象外）
- `trip list` / `show` / `stats` / `day list`（対象外）
- Travel Book export（対象外）
- write command / migration / FK（変更なし）
- itinerary detail 表示仕様（変更なし）
- JSON shape（変更なし）

---

## Lessons learned

- `itinerary show --json` は `ItineraryItem` のみ — service result も最小限で十分
- human 専用の予約一覧は CLI 層に残し、JSON 互換を守る
- `trip show` と同様の薄い service パターンが read-only show に有効
