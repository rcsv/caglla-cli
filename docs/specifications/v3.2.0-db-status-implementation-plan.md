## Purpose

```text
Caglla CLI が現在どの caglla.db を参照しているかを明示し、
サンプル DB・本番 DB・テスト DB の混同を防ぐ。
```

v3.2.0 では **パス変更機能は持ち込まない**。まず **解決済みパスの表示** と **ファイル存在・中身の確認** を固める。

## Current behavior（Phase 1 後）

| 項目 | Phase 1 後 |
|---|---|
| DB パス | 固定 `caglla.db`（実行時 CWD 基準）— `db path` で絶対パス表示 |
| 接続 | 大半のコマンドは `main.rs` 冒頭で `open_db()`。**例外:** `db path` / `db status` は open 前 dispatch |
| `db` サブコマンド | `db path` / `db status` / `db reset` |
| SQLite schema version | **専用列・PRAGMA なし** — `init_db` + migration 関数で段階追加 |
| trip export schema | `TRIP_EXPORT_SCHEMA_VERSION`（現行 **6**）— `db status` で `trip_export_schema_version` として表示 |

`db path` / 未存在 DB に対する `db status` は **open 前に分岐**し、SQLite ファイルを作成しない。

## Commands

### `caglla db path`

| 項目 | 仕様 |
|---|---|
| 出力 | 解決済み DB パスを **絶対パス** で 1 行表示 |
| 解決 | `std::env::current_dir()` + `DB_FILE`（現行と同じルール） |
| DB open | **しない** |
| ファイル未存在 | **作成しない** |
| `--json` | v3.2.0 では **不要**（テキストのみ） |

例:

```text
/Users/me/projects/caglla-cli/caglla.db
```

### `caglla db status`

人間向けテキストで、少なくとも以下を表示する。

| フィールド | 説明 |
|---|---|
| Path | 解決済み **絶対パス** |
| Exists | ファイルの有無（`true` / `false` または Yes / No） |
| File size | **存在時のみ** — バイト数 |
| Table counts | **存在時のみ** — 主要テーブル行数（下表） |
| Trip export schema | 現行 CLI の `TRIP_EXPORT_SCHEMA_VERSION`（**6**）— **SQLite migration version ではない** とラベルで明示 |

#### 存在しない場合

- DB を **open しない**
- ファイルを **作成しない**
- Path / Exists（false）/ Trip export schema（定数）まで表示し、size・counts は省略

#### 存在する場合

- 通常の DB 接続フロー（`open_db_at` → `init_db` / migration 適用）で open
- migration 適用 **後** の table counts を表示（他コマンドと同じ「実際に使われる DB 状態」）

#### Table counts 対象

| テーブル | 備考 |
|---|---|
| `trips` | |
| `days` | |
| `itinerary_items` | |
| `notes` | |
| `expenses` | |
| `expense_beneficiaries` | v3.0.0+ |
| `estimates` | v3.1.0+ |
| `participants` | v2.0.0+ |
| `reservations` | |
| `checklist_items` | |

存在しないテーブル（極古 DB）が query できない場合は、migration 適用後を前提とする（open 後は上表が揃う想定）。

### `caglla db status --json`

既存 read 系 `--json` と同様:

- stdout に **pretty JSON のみ**（人間向け見出しなし）
- envelope **`schema_version: 1`**（CLI JSON 出力フォーマット版。trip export schema とは別）

#### 提案フィールド

```json
{
  "schema_version": 1,
  "path": "/abs/path/caglla.db",
  "exists": true,
  "file_size_bytes": 204800,
  "trip_export_schema_version": 6,
  "table_counts": {
    "trips": 1,
    "days": 4,
    "itinerary_items": 12,
    "notes": 0,
    "expenses": 3,
    "expense_beneficiaries": 2,
    "estimates": 5,
    "participants": 2,
    "reservations": 0,
    "checklist_items": 8
  }
}
```

`exists: false` のとき:

- `file_size_bytes` / `table_counts` は **省略**（実装済み — `null` ではない）
- `trip_export_schema_version` は定数 **6** を含める（「現行 CLI が export する schema」）

**命名:** `trip_export_schema_version` = `models::TRIP_EXPORT_SCHEMA_VERSION`。envelope の `schema_version`（= 1）と混同しないこと。
