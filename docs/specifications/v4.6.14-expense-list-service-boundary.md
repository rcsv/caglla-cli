# Expense list service boundary

v4.6.13 `reservation list` に続き、**`expense list` 単独**で service 層を導入する。

| Item | Status |
|---|---|
| Target | v4.6.14 — expense list service boundary |
| Target command | **`expense list` のみ** |
| Scope | service 層導入（structured result）、CLI 出力互換維持 |

---

## Status

| 観点 | 判定 |
|---|---|
| service layer | **`src/services/expense_list.rs`** 追加 |
| target command | `expense list` が service 経由 |
| terminal I/O | service 内に `println!` / `eprintln!` なし |
| CLI human output | **不変**（テーブル形式） |
| CLI JSON output | **不変**（`ExpenseListJson` shape） |
| ordering | **不変** |
| trip stats | **変更なし** |
| Receipt / Estimate / Actual 境界 | **変更なし** |
| DB schema / migration | **変更なし** |

---

## Background

目標形:

```text
CLI / Tauri
  → service (use case, structured result, no I/O)
  → repository / domain
```

`expense list` は Trip / Itinerary 単位で Actual（Expense）一覧を表示する read-only 導線。`trip stats` の集計仕様には触れず、既存挙動を保ったまま service boundary のみ導入する。

---

## Target command

- `expense list --trip <id>`
- `expense list --itinerary <id>`
- `--json` フラグあり（`ExpenseListJson`）

`--trip` と `--itinerary` は同時指定不可。

---

## Current flow before split

```text
main.rs (ExpenseAction::List)
  → expense::resolve_expense_list_target(trip, itinerary)
  ├─ Trip      → list_expenses_for_trip
  └─ Itinerary → list_expenses_for_itinerary
  ├─ --json  → expense_to_json 各件 + ExpenseListJson
  └─ human   → print_expense_list(conn, target, &expenses)
```

---

## Service boundary introduced

新規:

```text
src/services/expense_list.rs
```

API:

```rust
pub struct ExpenseListServiceResult {
    pub target: ExpenseListTarget,
    pub expenses: Vec<Expense>,
}

pub fn list_expenses(
    conn: &Connection,
    trip: Option<i64>,
    itinerary: Option<i64>,
) -> Result<ExpenseListServiceResult>
```

- `ExpenseListJson` / `expense_to_json` は CLI 層に残す
- 新しい集計や total 表示は追加しない

---

## CLI compatibility

- human: `print_expense_list(&conn, result.target, &result.expenses)`
- JSON: `ExpenseListJson` を CLI 層で組み立てて serialize

---

## JSON compatibility

- `--json` 時は `ExpenseListJson`（`trip_id`, `itinerary_id`, `expenses: Vec<ExpenseJson>`）を serialize（shape 不変）

---

## Error compatibility

- target 未指定: `--trip または --itinerary のいずれかを指定してください`
- 同時指定: `--trip と --itinerary は同時に指定できません`
- Trip 不存在: `Trip not found: <id>`
- Itinerary 不存在: `Itinerary not found: <id>`

---

## Ordering compatibility

```text
ORDER BY itinerary_id ASC, sort_order ASC, id ASC
```

service は順序を変えない。

---

## trip stats / Actual boundary

| 概念 | 扱い |
|---|---|
| Expense | Actual の正本（変更なし） |
| Estimate | Planned（対象外） |
| Receipt Inbox | 未整理証憑候補（対象外） |
| trip stats | Expense のみを Actual として集計（変更なし） |

service は read-only list のみ。集計ロジックには触れない。

---

## Lessons learned

- Expense list は Trip / Itinerary とも同一テーブル human 表示 — `trip_context` 不要
- JSON は `expense_to_json` 変換が CLI 層に残る（participant 名解決等）
- trip stats との境界を守るため、service は list のみに限定
