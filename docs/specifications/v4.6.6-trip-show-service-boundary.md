# Trip show service boundary

v4.6.4 (`trip stats`) / v4.6.5 (`trip list`) に続き、read-only command の service boundary を **`trip show` 単独**で導入する。

| Item | Status |
|---|---|
| Target | v4.6.6 — trip show service boundary |
| Target command | **`trip show` のみ** |
| Scope | service 層導入（structured result）、CLI 出力互換維持 |

---

## Status

| 観点 | 判定 |
|---|---|
| service layer | **`src/services/trip_show.rs`** 追加 |
| target command | `trip show` が service 経由 |
| terminal I/O | service 内に `println!` / `eprintln!` なし |
| CLI / JSON 出力 | **不変** |
| not found | **不変**（`Trip not found: <id>`） |
| DB schema / migration | **変更なし** |

---

## Background

目標形:

```text
CLI command
  → service/usecase
  → repository
  → SQLite

Tauri command
  → same service/usecase
  → repository
  → SQLite
```

`trip show` は GUI の詳細画面に直結するため、service boundary の次段階として扱う。ただし v4.6.6 では拡大しすぎないため、対象は `trip show` のみに限定する。

---

## Target command

- `trip show <trip_id>`
- `trip show <trip_id> --json`

---

## Current flow before split

```text
main.rs (TripAction::Show)
  → trip::get_trip(conn, id) -> Trip
  ├─ --json  → print_json(&Trip)
  └─ human   → trip::print_trip_detail(&Trip)
```

---

## Service boundary introduced

新規:

```text
src/services/trip_show.rs
```

API:

```rust
pub struct TripShowServiceResult {
    pub trip: Trip,
}

pub fn show_trip(conn: &Connection, trip_id: i64) -> Result<TripShowServiceResult>
```

service は `trip::get_trip` を呼び、**structured result を返すのみ**（terminal I/O なし）。

---

## CLI compatibility

- human 出力は従来どおり `trip::print_trip_detail(&trip)` を使用
- handler は service 経由に変更するが、表示仕様は変更しない

---

## JSON compatibility

- `trip show --json` は引き続き `Trip` をそのまま serialize（object shape / field は不変）

---

## Error compatibility

not found 時:

- 従来どおり `Trip not found: <id>` を返す（exit / stderr の挙動は既存テストで担保）

---

## Test coverage

- service test:
  - 既存 Trip を返す
  - not found 時の error message を維持する
- 既存 CLI の not-found テスト（`tests/not_found_cli.rs`）を回帰として維持

---

## What changed

- `src/services/trip_show.rs` 追加
- `src/services/mod.rs` に `trip_show` module 追加
- `main.rs` の `TripAction::Show` が service 経由に

---

## What did not change

- `day list` / `itinerary list` / `itinerary timeline`（対象外）
- `trip list` / `trip stats`（対象外）
- write command / import / duplicate / delete / receipt assign（対象外）
- DB schema / migration / FK（変更なし）
- CLI 出力仕様 / JSON schema（変更なし）

---

## Lessons learned

- `trip show` も既存 `Trip` を返しており、service は thin wrapper で十分
- `--json` は service に入れず、CLI が出力モードを選ぶ方針を維持できた
