# v4.4.2 — Travel Book presentation helper review

Caglla.Travel CLI **v4.4.1** 時点の `src/io/markdown.rs` を棚卸しし、**renderer 非依存の presentation ルール**と **Markdown renderer 詳細**を分類するレビューです。

| ドキュメント | 役割 |
|---|---|
| [v4.4.0-travel-book-presentation-model-review.md](v4.4.0-travel-book-presentation-model-review.md) | 三層アーキテクチャ・全体方針 |
| [v4.4.1-category-display-name-in-travel-book-implementation-plan.md](v4.4.1-category-display-name-in-travel-book-implementation-plan.md) | 最初の helper 抽出（category） |
| **本書** | **v4.4.2** — `markdown.rs` 関数単位の棚卸し（**documentation-only**） |

検証フィクスチャ: [samples/okinawa_sesoko_2026/expected-export-md.md](../../samples/okinawa_sesoko_2026/expected-export-md.md)

現行実装: [src/io/markdown.rs](../../src/io/markdown.rs)

## 1. Summary

v4.4.1 で `format_travel_book_category_detail_line` が **presentation 寄り helper** として切り出された。残りの `markdown.rs` は依然として **章組み立て・表示判断・Markdown 構文**が同居している。

**結論:**

1. **約 60% の Travel Book 判断は presentation helper 候補** — GUI / Web でも再利用したい
2. **Markdown 構文は局所的** — 見出し級・表・checkbox・余白は renderer に留める
3. **v4.4.2 では移動しない** — 地図を作る。抽出は v4.4.3 以降の小さな単位で
4. **v4.4.1 の category helper は正しい方向** — 同パターンで reservation / notes / overview ルールを段階抽出可能

## 3. Chapter assembly（全体）

`format_trip_markdown` の章順序と空章省略:

```text
Cover → Trip overview（条件付き）→ Daily schedule → Reservations? → Checklist? → Planned cost? → Notes? → Colophon
```

| 判断 | 分類 | 関数 / 箇所 |
|---|---|---|
| 章の順序（v4.1.0 正本） | **B** | `format_trip_markdown` |
| Trip overview 章の有無 | **B** | `trip_overview_worth_showing` |
| Reservations / Checklist / Planned cost / Notes 0 件時省略 | **B** | 各 `format_*` の `Option` 返却 |
| Colophon は常に末尾 | **B** | `append_colophon` |
| 章見出し `## Trip overview` 等 | **C** | 各 `append_*` / `format_*` |

## 4. Review point 1 — Trip overview metrics

### 4.1 表示可否

| 判断 | 分類 | 実装 |
|---|---|---|
| summary / participants / stats のいずれかがあれば Trip overview 章を出す | **B** | `trip_overview_worth_showing` |
| Trip summary 空文字は無視 | **B** | `append_trip_overview` |
| Participants 表の有無 | **B** | `!participants.is_empty()` |
| Cover と Trip overview の Travelers 重複（Cover は簡易、overview は詳細） | **B** | `append_cover` vs `append_trip_overview_stats` |

### 4.2 メトリクス行

| 判断 | 分類 | 実装 |
|---|---|---|
| Days / Itineraries / Checklist 件数 | **A**（値）+ **B**（ラベル・出し分け） | `TripStats` + `append_trip_overview_stats` |
| Stay / Travel / Total が **すべて 0** なら 3 行省略 | **B** | `append_trip_overview_stats` L116–129 |
| 時間の人間可読化（`2h 30m` 等） | **B** | `format_minutes_duration`（`statistics.rs`） |
| メトリクス行の `- Days:` 箇条書き | **C** | Markdown bullet |
| Participants 表 `\| # \| Name \|` | **C** | pipe table |
| フィールドラベル `Days:` / `Stay Time:`（英語） | **B** | 将来 i18n 対象 — **E**（v4.4.x では触らない） |

### 4.3 Okinawa 観測

沖縄 golden では Stay / Travel / Total 行は出ない（全ゼロ省略が効いている）。Days / Itineraries / Checklist は表示。

## 13. Recommended extraction order（v4.4.3 候補）

大規模 rewrite はしない。**既に pub(crate) の関数をモジュールへ移す**のが最小コスト。

### Phase 1（v4.4.3 推奨）

```text
src/travel_book/presentation.rs（新規モジュール名は実装時確定）
  - reservation_provider_line_redundant          （移動）
  - format_travel_book_reservation_period      （移動）
  - format_travel_book_reservation_heading     （ロジック部分）
  - format_travel_book_category_detail_line    （移動）
  - travel_book_note_sort_key                  （移動）
```

`markdown.rs` はこれらを import して Markdown 構文だけ担当。

### Phase 2（v4.4.4+ 候補）

```text
  - trip_overview_worth_showing
  - append_days_overview のデータ組み立て（Vec<DayOverviewEntry>）
  - planned cost has_any_note 判定
```

### Phase 3（Defer）

```text
  - build_travel_book_document 全体
  - field label i18n
  - GUI renderer
```

## 14. v4.4.3 vs Defer — decision guide

| 進める条件 | 内容 |
|---|---|
| **v4.4.3 へ** | Phase 1 のみ — ファイル移動 + 既存テスト維持、golden 不変 |
| **Defer** | `TravelBookDocument` 型導入、章 renderer 完全分離、GUI 着手 |
| **別 milestone** | Trip overview 英語ラベル i18n、Daily schedule cue |

沖縄 golden が変わらない移動は **低リスク**。表示ルール変更を伴う抽出は **別バージョン**に分離する。
