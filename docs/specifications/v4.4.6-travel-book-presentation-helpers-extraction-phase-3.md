# v4.4.6 — Travel Book presentation helpers extraction Phase 3 (implementation)

Phase 3 extraction from the v4.4.5 extraction review §5 (R1–R4).

| Item | Status |
|---|---|
| Release | v4.4.6 |
| Scope | Label / context helper extraction only — no output behavior change |
| Validation fixture | [Okinawa Sesoko 2026 sample](../../samples/okinawa_sesoko_2026/README.md) |

## Goal

Move remaining light presentation label / context construction from `markdown.rs` into `travel_book_presentation.rs`.

## New helpers

| Function | Role |
|---|---|
| `travel_book_trip_date_range` | Trip date range `start 〜 end` |
| `travel_book_note_heading_label` | Note heading label (`Trip —` / `Day N —` / `Day N / … —`) |
| `travel_book_note_itinerary_context` | Itinerary-scoped note context (`Day N / time title`) |
| `travel_book_itinerary_context_label` (private) | Shared `Day N / …` pattern for notes and planned cost |

## Reused helpers

| Function | Role |
|---|---|
| `travel_book_day_overview_label` | Daily schedule `###` label body (`Day N — date`) — same as Days overview |

## Stays in `markdown.rs`

Markdown renderer concerns: `###` / `####` headings, tables, bullets, blank lines.

Thin wrappers `format_day_heading` / `format_note_heading` prepend `### ` only.

## Non-goals

- `TravelBookDocument` abstraction
- Output / golden body changes (Colophon `Version:` only)
- DB / JSON export schema changes
