# v3.9.0 — Config and DB Path Foundation Implementation Plan

Caglla.Travel CLI **v3.9.0** で **Config and DB path foundation**（DB 参照先の明示的解決）を追加する実装計画です。

| 項目 | 内容 |
|---|---|
| リリース | v3.9.0 — Config and DB path foundation |
| 前提 | v3.8.0（Roadmap realignment）リリース済み |
| Phase 1 | **実装対象** — 本書のスコープ |

---

## Purpose

```text
Caglla が参照する SQLite DB パスを、CLI / 環境変数 / ローカル config から明示的に解決できるようにする。
v3.2.0 の db path / db status を拡張し、通常コマンドも同じ解決結果を使う。
```

---

## Resolution priority（Phase 1）

```text
1. --db <path>          （CWD 基準の相対パス）
2. CAGLLA_DB            （CWD 基準）
3. ./caglla.toml        [database].path （相対パスは config ファイルのディレクトリ基準）
4. ./caglla.db          （CWD 基準、従来 default）
```

---

## caglla.toml（Phase 1）

- 読み込み対象は **CWD の `./caglla.toml` のみ**
- 親ディレクトリ探索なし
- ユーザーグローバル config なし
- config ファイルの自動生成・更新なし

最小フォーマット:

```toml
[database]
path = "./caglla.db"
```

- invalid TOML / empty `[database].path` はエラー

---

## CLI

- root `Cli` に global `--db <path>` を追加
- `caglla --db ./x.db <command>` および `caglla <command> --db ./x.db`（clap `global = true`）

---

## Implementation modules

| モジュール | 責務 |
|---|---|
| `src/config.rs` | `ResolvedDbPath`, `DbPathSource`, `resolve_db_path`, TOML 読み込み |
| `src/storage/db.rs` | `open_db_at`, `collect_db_status`, `db status` JSON v2 |
| `src/main.rs` | parse 後に一度 resolve → `run_before_open_db` / `open_db_at` |
| `src/commands/db.rs` | open 前後 dispatch、`db reset` 出力 |

`open_db()` の `caglla.db` 固定依存は通常コマンド経路から除去。

---

## db path / db status / db reset

| コマンド | 挙動 |
|---|---|
| `db path` | resolved path を表示。DB を作成しない |
| `db status` | resolved DB の状態。未存在時は open しない |
| 通常コマンド | resolved DB を open し、必要なら初期化 |
| `db reset` | resolved DB を初期化。出力に `DB: <path>` |

### db status human 出力（追加）

```text
Path source               : default
```

### db status JSON（schema v2）

additive フィールド:

```json
{
  "path_source": "default",
  "config_path": null
}
```

trip export schema は変更しない。

---

## Non-goals（Phase 1 で実装しない）

```text
- db use
- caglla.toml の自動生成・更新
- ユーザーグローバル config
- 親ディレクトリ探索
- profile 切替
- remote DB / cloud sync
- SQLite schema 変更
- trip export schema 変更
- Receipt / Expense lifecycle 変更
```

## Tests

### Resolver unit tests（`src/config.rs`）

- CLI > env > config > default
- config 相対パスは config dir 基準
- invalid TOML / empty path は error
- env は `DbPathResolveInputs` 注入（並列テスト安全）

### Integration tests（`tests/db_cli.rs`）

- `--db`, `CAGLLA_DB`, `caglla.toml`
- `db status` 未存在 DB 非作成
- 通常コマンドは selected DB のみ作成
- default `caglla.db` 既存テスト維持
