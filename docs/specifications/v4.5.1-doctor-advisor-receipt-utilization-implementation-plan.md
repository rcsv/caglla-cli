# v4.5.1 — doctor / advisor Receipt utilization (implementation)

v4.5.0 responsibilities review の結論に従い、Receipt Inbox を **Actual として集計せず**、ledger 整理（hygiene）のための **注意・助言** として `trip doctor` / `trip advisor` から可視化する。

| Item | Status |
|---|---|
| Release | v4.5.1 |
| Scope | doctor/advisor の issue/advice 追加（stats/export-md/export schema は不変） |
| Validation fixture | [Okinawa Sesoko 2026 sample](../../samples/okinawa_sesoko_2026/README.md) |

---

## Goal

- Trip に pending receipt が残っていることを doctor/advisor で検出できる
- Receipt の状態矛盾（`status` と `trashed_at` の不整合）を warning として検出できる
- advisor は `--with-commands` で実在する CLI コマンド例を提示できる

**Core principle（維持）**

```text
Receipt Inbox = 未整理証憑候補（Actual ではない）
Expense       = Actual の正本
Estimate      = Planned
trip stats    = Expense のみを Actual として集計
Travel Book   = Receipt / Expense / Difference を含めない
```

---

## Changes

### 1) Doctor issues

| Issue | Condition | Severity | Target |
|---|---|---|---|
| `pending_receipts` | `trashed_at IS NULL` の Receipt が 1 件以上 | **info** | Trip |
| `inconsistent_receipt_state` | `ignored && trashed_at IS NULL` または `unreviewed && trashed_at IS NOT NULL` | **warning** | Receipt(id) |

Active（pending）の定義は **`trashed_at IS NULL`**（status は通常 `unreviewed`）。

### 2) Advisor advice / commands

pending receipts がある場合:

- Advice: inbox の確認 → `assign` で Expense 化、または `trash`
- `--with-commands`:

```bash
cargo run -- receipt list --trip <trip_id>
cargo run -- receipt assign <receipt_id> --itinerary <itinerary_id>
cargo run -- receipt trash <receipt_id>
```

inconsistent receipt state がある場合（状態矛盾の解消導線）:

```bash
cargo run -- receipt list --trip <trip_id> --all
cargo run -- receipt trash <receipt_id>
cargo run -- receipt restore <receipt_id>
```

### 3) JSON output

- `DoctorIssueCode` に `pending_receipts` / `inconsistent_receipt_state` を追加
- `DoctorIssueTarget` / `IssueTargetType` に `Receipt(id)` / `receipt` を追加
- `details` に `receipt_count` / `receipt_id` を追加

#### Schema version

doctor/advisor JSON envelope は `schema_version: 2` に bump（target type / details fields の拡張を明示するため）。

## Non-goals

```text
- trip stats の Actual 定義変更（Receipt/Pending を含めない）
- Travel Book export-md への Receipt 追加
- export/import schema / SQLite schema 変更
- Receipt assign 消費モデル変更（assign 後 Receipt は削除）
- Potential Actual の表示（stats/export-md）
- Attachment / Photo / Travel Journal
```

## Tests

- doctor:
  - pending receipts がある Trip で info issue
  - trashed receipts のみでは pending issue なし
  - inconsistent receipt state がある場合 warning issue
- advisor:
  - pending receipts の advice
  - `--with-commands` で receipt 系コマンドが含まれる
- JSON:
  - doctor/advisor の envelope `schema_version = 2`
