## 1. Summary

v3.6.0 は **Receipt Inbox metadata-only support** を実装した機能候補リリースです。Receipt は **Expense 化待ちの未整理支払い候補** として Trip 直下に保存され、**Actual spending ではない** レイヤーとして扱います。

### 到達点

| 項目 | 判定 |
|---|---|
| Receipt Inbox metadata-only 実装 | ✅ |
| Receipt = Trip-level collection（Itinerary 配下にネストしない） | ✅ |
| status: `unreviewed` / `ignored` のみ | ✅ |
| Planned vs Actual / trip stats / export-md 非含有 | ✅ |
| export schema **v7**（`trip.receipts[]`） | ✅ |
| v6 import 互換維持 | ✅ |
| `image_path` / Receipt image 未実装 | ✅ |
| `receipt link` 削除、`receipt convert` deferred | ✅ |
| **`make check`** | ✅ |

**結論:** Receipt Inbox は metadata-only として実装完了。release 前に lifecycle を単純化し、GUI / CUI 実運用に合う設計に収束した。Release blocker となる実装問題はない。

## 3. Final lifecycle design

release 前（`796b894`）に、GUI / CUI 実運用を踏まえて **Receipt status を単純化** した。

### 最終 user-facing status

```text
unreviewed
ignored
```

| status | Meaning |
|---|---|
| `unreviewed` | Expense 化されていない未整理支払い候補 |
| `ignored` | 旅行費用ではない、重複、対象外 |

### 削除・deferred にしたもの

```text
linked status
converted status
receipt link command
itinerary_id
linked_expense_id
receipt convert / promote
```

### 判断理由

| 判断 | 理由 |
|---|---|
| `linked` / `receipt link` 削除 | Itinerary にだけ紐づけて Actual にしない状態は中途半端で、運用が分かりにくい |
| `converted` 削除 | Expense 化済み Receipt を Inbox 上で別 status にする必要性が低い（metadata-only では証憑として残す意味も薄い） |
| Itinerary 判明時は Expense 昇格 | GUI では Itinerary 選択 = Expense 昇格が自然。`receipt convert` / `promote` は future work |
| 画像証憑は Evidence / Attachment | Receipt 専用 `image_path` ではなく、Receipt / Expense 共通レイヤーで将来検討 |

## 4. Data model review

### `receipts` table（最終形）

```sql
CREATE TABLE receipts (
  id                  INTEGER PRIMARY KEY AUTOINCREMENT,
  trip_id             INTEGER NOT NULL,
  day_id              INTEGER NULL,
  amount              INTEGER NULL,
  currency            TEXT NULL,
  occurred_date       TEXT NULL,
  memo                TEXT NULL,
  status              TEXT NOT NULL,
  created_at          TEXT NOT NULL,
  updated_at          TEXT NOT NULL
);
```

インデックス: `idx_receipts_trip`、`idx_receipts_day`。

### フィールド整理

| フィールド | 方針 | 判定 |
|---|---|---|
| `id` | 内部 DB id（export しない） | ✅ |
| `trip_id` | **必須** — Receipt は Trip に属する | ✅ |
| `day_id` | **optional** — 「この日に発生したっぽい」ヒント | ✅ |
| `amount` / `currency` | optional、**ペア必須** | ✅ |
| `occurred_date` | optional — ISO 日付文字列 | ✅ |
| `memo` | optional（実用上重要） | ✅ |
| `status` | `unreviewed` / `ignored` | ✅ |
| `created_at` / `updated_at` | 必須 | ✅ |
| `itinerary_id` | **持たない** | ✅ |
| `linked_expense_id` | **持たない** | ✅ |
| `image_path` | **持たない** | ✅ |

### 階層・cascade

| 観点 | 挙動 | 判定 |
|---|---|---|
| Trip 階層 | Receipt は **Trip-level collection**。Itinerary 配下には置かない | ✅ |
| `trip delete` | 当該 Trip の receipts を削除 | ✅ |
| `day delete` | Receipt は **残す**。`day_id` を NULL に（`nullify_receipts_for_day`） | ✅ |
| FK | DB FK なし + アプリ側検証（Expense / Estimate と同型） | ✅ |

実装: [src/receipt.rs](../../src/receipt.rs)、[src/domain/models.rs](../../src/domain/models.rs)。

### `ExportReceiptV7` 構造

```json
{
  "day_ref": { "day_number": 1 },
  "amount": 1700,
  "currency": "JPY",
  "occurred_date": "2026-04-26",
  "memo": "これなんだっけ？",
```

含めないキー: `itinerary_ref`、`linked_expense_ref`、`image_path`。

### import / validate-export / diff

| `trip validate-export` | `Receipts` check + `collect_export_receipt_validation_errors` | ✅ |
| `trip diff` | receipt added / removed / modified（v7 同士） | ✅ |
| legacy status import | `linked` / `converted` → `unreviewed` に正規化 | ✅ |
| validate-export | `linked` / `converted` は **invalid** | ✅ |

実装: [src/trip.rs](../../src/trip.rs)、[src/io/diff.rs](../../src/io/diff.rs)。

## 7. Planned vs Actual relationship

```text
Estimate → Planned
Expense  → Actual
Receipt  → Expense 化待ちの未整理支払い候補（Actual ではない）
```

Receipt は **Actual spending として確定していない**。Planned vs Actual difference（`Actual − Planned`）には **含めない**。

### 変更していないもの

| 機能 | 判定 |
|---|---|
| `trip export-md` Planned / Actual / Difference | ✅ 変更なし |
| `trip stats` | ✅ 変更なし |
| `trip stats --json` | ✅ 変更なし |
| Day-level Planned vs Actual | ✅ 変更なし |
| Itinerary-level difference calculation | ✅ 変更なし |

### 確認済みテスト

| テスト | 内容 |
|---|---|
| `cli_receipt_does_not_affect_trip_stats` | Receipt 追加後も `expense_count` / `expense_totals` 不変、`receipt_count` キーなし |
| `test_stats_*` / `test_export_md_*` | 既存 Planned vs Actual 回帰テスト群（`make check` で PASS） |

## 8. Image / evidence handling

### 今回の Non-goals（未実装）

```text
image_path
receipt image support
expense image support
image binary storage
OCR
automatic receipt parsing
generic Attachment / Photo
Evidence / Attachment table
```

### 将来方針

画像証憑は **Receipt 専用ではなく**、**Receipt / Expense の両方に付与できる Evidence / Attachment 共通レイヤー** として検討する。ビジネストリップや経費精算では確定済み Expense にも証憑が必要になるため、Receipt にだけ `image_path` を持たせない。

## 9. Validation review

| ルール | 実装 | 判定 |
|---|---|---|
| status は `unreviewed` / `ignored` のみ | `validate_receipt_status` | ✅ |
| amount / currency ペア必須 | `validate_receipt_amount_currency_pair` | ✅ |
| amount のみ / currency のみはエラー | add / update / CLI テスト | ✅ |
| amount / currency なしの場合は memo 必須 | `validate_receipt_has_content` | ✅ |
| currency rule | Expense / Estimate と同じ（`money.rs`） | ✅ |
| negative decimal 表示 | `format_amount_optional` — USD `-0.50` 等 | ✅ |
| invalid day reference | 同一 Trip 外 Day は拒否 | ✅ |
| `ignored` でも amount / currency / memo 保持 | `receipt ignore` + CLI テスト | ✅ |
| `image_path` | 受け付けない（フィールド・CLI フラグともに不存在） | ✅ |
| `occurred_date` | Expense と同型の日付検証 | ✅ |
| export validate | status / amount-currency / memo-or-amount / currency / occurred_date | ✅ |
