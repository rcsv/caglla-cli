# v4.6.0 — TripStats.days semantics fix

Core architecture hardening の最初の具体修正。`TripStats.days` の意味を **Trip day count** に正す。

| Item | Status |
|---|---|
| Release | v4.6.0 |
| Scope | `compute_trip_stats` の `days` 算出のみ（DB schema / export schema 不変） |
| Validation fixture | [Okinawa Sesoko 2026 sample](../../samples/okinawa_sesoko_2026/README.md) |

---

## Problem

従来の `TripStats.days` は `itinerary_items` の最大 `day` から算出されていた。

```text
Trip: 2026-04-26 〜 2026-04-29 = 4日間
Itinerary: Day 1 のみ登録
→ days == 1（誤り）
```

`trip stats` と Travel Book `Trip overview` の `- Days:` は **旅行日数** を表すべきであり、Itinerary の登録状況に引っ張られてはならない。

---

## Goal

```text
TripStats.days = Trip day count
Not: max itinerary day
```

`TripStats.days` は **出発日〜帰着日の日数**（`trip_day_count`）を表す。日本語の「何泊何日」表記そのものではない（後述）。

### Resolution order

1. **`days` テーブル件数**（正本）
2. `days` が 0 件かつ Trip に有効な `start_date` / `end_date` がある → **日付範囲から算出**
3. それも不可 → **legacy fallback** として itinerary max day

通常の現行 DB では Trip 作成時に `days` が作成され、migration でも backfill されるため、基本は `days` テーブル由来。

---

## Related concepts: `days` vs `nights` vs `duration_label`

`TripStats.days` の修正と、日本語の「日帰り」「1泊2日」「3泊5日」は **近いが同じではない**。v4.6.0 では `days` の意味を正すことに集中し、宿泊数・表示ラベルは defer する。

| Concept | Meaning | v4.6.0 |
|---|---|---|
| `trip_day_count` | 出発日〜帰着日の日数 | **`TripStats.days` がこれ** |
| `accommodation_night_count` | 宿泊施設に泊まる夜数 | **未実装・defer** |
| `duration_label` | locale / presentation 用の表示文字列 | **未実装・defer** |

`days` と `nights` は別概念。普通の旅行では `nights = days - 1` になりがちだが、飛行機内泊や夜行移動を含む場合は一致しない。

```text
1 day, 0 nights  → 日帰り
2 days, 1 night  → 1泊2日
4 days, 3 nights → 3泊4日
5 days, 3 nights → 3泊5日
```

**v4.6.0 では `nights` を `days - 1` として DB に決め打ちしない。**

将来、Travel Book presentation または GUI 表示設計のタイミングで、例えば次のような helper を検討できる:

```text
travel_duration_label(locale, trip_day_count, accommodation_night_count)
```

日本語表示例:

```text
days = 1, nights = 0  → 日帰り
days = 2, nights = 1  → 1泊2日
days = 5, nights = 3  → 3泊5日
nights が不明       → 5日間
```

## Implementation

### `trip_day_count_for_stats`

`src/analysis/statistics.rs` に helper を追加し、`compute_trip_stats` から利用。

### `TripStats.days` コメント

```rust
/// Trip に属する日数（出発日〜帰着日、`trip_day_count`）。Itinerary の最大 day でも、宿泊数（nights）でもない。
pub days: i64,
```

### CLI / JSON

- Human output: field 名 `Days` 維持
- `trip stats --json`: field 名 `days` 維持（意味のみ修正）

Release notes 記載:

```text
trip stats `days` now represents Trip day count, not max itinerary day.
```

## Tests

| Case | Expected |
|---|---|
| 4日 Trip + Day 1 itinerary のみ | `stats.days == 4` |
| 3日 Trip + itinerary 0 件 | `stats.days == 3` |
| `days` 削除 + 日付あり | 日付範囲 fallback → 4 |
| `days` なし + 日付なし + itinerary day 2 | legacy max day → 2 |
| 既存 stats（expense / estimate / checklist 等） | 回帰なし |
| Okinawa sample | `Days: 4`（本文 golden 不変、Colophon Version のみ更新） |
