# Read-only service boundary expansion

v4.6.4 の `trip stats` pilot に続き、read-only command を **service boundary** 経由にする取り組みを `trip list` に横展開する。

| Item | Status |
|---|---|
| Target | v4.6.5 — read-only service boundary expansion |
| Target command | **`trip list` のみ** |
| Scope | service 層導入（structured result）、CLI 出力互換維持 |

---

## Status

| 観点 | 判定 |
|---|---|
| service layer | **`src/services/trip_list.rs`** 追加 |
| pilot command | `trip list` が service 経由 |
| terminal I/O | service 内に `println!` / `eprintln!` なし |
| CLI / JSON 出力 | **不変** |
| DB schema / migration | **変更なし** |

---

## Background

目標形:

```text
CLI command
  → service/usecase
  → repository
  → SQLite

Tauri command
  → same service/usecase
  → repository
  → SQLite
```

v4.6.5 は拡大しすぎないため、対象は `trip list` のみに限定する。

---

## Target command

- `trip list`
- `trip list --json`

---

## Current flow before split

```text
main.rs (TripAction::List)
  → trip::list_trips(conn) -> Vec<Trip>
  ├─ --json  → print_json(&Vec<Trip>)
  └─ human   → trip::print_trip_list(&Vec<Trip>)
```

---

## Service boundary introduced

新規:

```text
src/services/trip_list.rs
```

API:

```rust
pub struct TripListServiceResult {
    pub trips: Vec<Trip>,
}

pub fn list_trips(conn: &Connection) -> Result<TripListServiceResult>
```

service は `trip::list_trips` を呼び、**structured result を返すのみ**（terminal I/O なし）。

---

## CLI compatibility

- human 出力は従来どおり `trip::print_trip_list(&trips)` を使用（見出し・空表示・合計表示を含む）
- ordering は従来どおり `trip::list_trips` の `ORDER BY id`

---

## JSON compatibility

- `trip list --json` は引き続き `Vec<Trip>` のまま serialize（配列 shape / field は不変）

---

## Test coverage

- service test:
  - empty DB → empty list
  - 2 trips 追加 → id 順を維持
- 既存 CLI テストは回帰として維持

---

## What changed

- `src/services/trip_list.rs` 追加
- `main.rs` の `TripAction::List` が service 経由に

---

## What did not change

- `trip show` / `day list` / `itinerary list` / `itinerary timeline` / `trip stats`（対象外）
- write command / import / duplicate / delete / receipt assign（対象外）
- DB schema / migration / FK（変更なし）
- CLI 出力仕様 / JSON schema（変更なし）

---

## Lessons learned

- `trip list` は既に `Vec<Trip>` を返しており service は thin wrapper で十分
- `--json` は service に入れず、CLI が出力モードを選ぶ方針を維持できた
